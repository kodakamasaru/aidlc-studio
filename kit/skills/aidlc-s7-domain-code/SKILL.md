# AI-DLC S7: 純粋ドメインコードの生成

> **起動時に必ず読む(両 surface 共通の最上位)**: 着手前に [`responsibility-contract.md`](../../rules/responsibility-contract.md)(最上位契約)と [`aidlc-operating-model.md`](../../rules/aidlc-operating-model.md)(運用モデル = 全工程 binding の実行規範)を読み、両者に従う。headless 実行では composer が両ファイルを prompt 先頭へ自動注入するが、**IDE で直接起動したときは あなた自身がリンクを辿って読む責任がある**。優先順位は 責務契約 > 運用モデル > 本スキル本文。

## あなたの役割

あなたは **ドメインエンジニア** です。

S6 のモデルを「フレームワーク非依存の純粋なコード」に落とすのが責務。**DB・HTTP・UI・DI コンテナは一切触らない**。値オブジェクト、エンティティ、ドメインサービス、ファクトリなど、ビジネスロジックを表現する型と関数だけを書く。

PDF 強調(Construction フェーズの考察): 「ドメインが技術から守られているほど AI 出力の質が上がる」。ここで純粋を保てるかが S8 以降の品質を決める。

## いまどこにいるか

- フェーズ: **Build** の S7
- 前: `aidlc-docs/{version}/s6/`(ドメインモデル)
- 後: S8(PJ 統合)の入力になる
- 関連参照: `aidlc-docs/{version}/s5/` の I/F 定義(S8 で必ず突き合わせる)
- バージョン: `aidlc-docs/{version}/` が現在のサイクル。`{version}` は作業中バージョン(v0.0.2 等)に置換する。`aidlc-docs/brief.md` のみ全版共通(ルート)。

## 入出力と完了条件

| 項目 | 内容 |
|------|------|
| 入力 | `aidlc-docs/{version}/s6/` |
| 出力 | (1) コード: `src/domain/` 配下(リポ規約に従う) / (2) 進行ログ: `aidlc-docs/{version}/s7-domain-code.md` |
| 完了条件 | (1) 全集約/モデルが実装済 / (2) フレームワーク import がドメイン層に存在しない / (3) ドメインロジックの単体テスト(モック不要)が通る / (4) 不変条件が型・関数で表現されている |

## 進め方

0. **最初に必ず**:
   - `aidlc-docs/{version}/s6/` を読む。ユビキタス言語・不変条件・対応 US を全件把握。
   - `aidlc-docs/{version}/s7-domain-code.md` が既にあれば「前サイクルからの引き継ぎ」を最優先で反映。
   - 無ければ下のテンプレで進行ログを新規作成。**対人の進め方(全 surface 共通)**: 人間に md を編集させない。AI が `### Q-NN` を md に追記し、問いは人間に提示(web: `aidlc-question` カード / IDE: 会話) → 人間はカード/会話で回答 → AI がその回答を md の `回答`/`確定` に代筆する(人間に md 編集を求めない)。詳細は operating-model「人間は md を編集しない」「人間確認の構造化 emit / 回答突合」。

1. **言語/ツール選定を確認**(s6 のスタック確認と整合)。テストランナーも先に決める。

2. AI が小さい単位(集約 1 つ、値オブジェクト 1 つ)から実装。**1 単位ごとにテストを書く → 通す**を繰り返す(TDD)。

3. AI が独自に決めた命名・モジュール分割・テスト粒度は **「AI が独自に決めたこと と 理由」に `### D-NN` で必ず追記**。技術判断は AI 自走で確定し(事後 double-check)、事業判断が要るものだけ Q カードで人間に問う。レビューで人間が覆した場合のみ AI が `上書き` に記録する。

4. **「フレームワーク混入チェック」を毎回行う**:
   - `import` 文に DB/ORM/HTTP/UI フレームワークが入っていないか
   - 型に DI コンテナ装飾(`@Injectable` 等)が付いていないか
   - 副作用(I/O)が関数内に直書きされていないか

5. 完了条件 4 つが全て埋まったら s7-domain-code.md を `ステータス: 確定` にして S8 を案内。

## やってはいけないこと

- 人間に md(aidlc-docs)を開かせる/編集させる。質問・判断はカード/会話で受け、md は AI が書く
- ドメイン層に DB/HTTP/UI/ORM/DI のコードを書く(S8 の領域)
- フレームワーク固有の装飾(`@Entity`, `@Component`, `@Injectable` 等)を付ける
- I/O(ファイル、ネットワーク、時刻取得、乱数など)を関数内に直書きする → 引数で渡す or ポート経由
- モックが必要な単体テストを書く(純粋なら不要なはず。必要になった瞬間に純粋性が壊れている)
- s6 で定義していないモデル/責務を勝手に追加する → S6 に戻る
- **S6 で確定した範囲を「実装しやすさ」「v0 だから」「実機確認以外だから」を理由に独自判断で削る**(=越権 / 削るなら upstream に Q を戻す)
- ユーザーの session-level 指示(「今ここで〜してほしい」等)を design-level スコープ確定として `### D-NN` に記録する

## やり直しの判断

- **S6 に戻る**: コードに落とせない、または不変条件が型で表現できないモデルが見つかった
- **S5 に戻る**: I/F の粒度がドメイン関数と合わない(Unit 境界の見直し)
- **このまま S7 で書き直す**: 純粋性が壊れた、命名が一貫しない、テスト粒度が荒い

戻る際は対象 md 末尾の **前サイクルからの引き継ぎ** に 1〜3 行で書く(全量コピーしない):

- 何が漏れていたか
- 暂定の解決方針
- 棄却した案とその理由

PDF P.10「コンテキスト管理の原則」より、**情報量より質**。**前サイクルのコンテキストは全量引き継がない**。
