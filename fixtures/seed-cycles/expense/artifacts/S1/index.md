## 全体方針

個人の日常支出を記録・把握するシンプルな家計簿アプリ。サーバーレス(IndexedDB ローカル永続)・シングルユーザーを前提とし、MVP では記録/削除/一覧/月次集計/予算の5軸を最小限の画面で実現する。複雑なタグ・繰り返し入力・クラウド同期はスコープ外。

### US-01: 支出記録

**As a** 家計管理をしたい個人ユーザー  
**I want to** 金額・日付・カテゴリ・メモを入力して支出1件を登録したい  
**So that** 日々の支出を手軽にトラッキングできる

**受け入れ条件:**
- 金額(正整数・円単位)、日付、カテゴリ(固定6種)、メモ(任意・最大200文字)を入力できる
- 登録後、直近一覧に即時反映される
- 金額が0以下の場合はバリデーションエラーを表示する
- メモが200文字を超える場合はバリデーションエラーを表示する

### US-02: 支出削除

**As a** 誤入力を修正したい個人ユーザー  
**I want to** 登録済みの支出を削除したい  
**So that** 誤った記録を取り除ける

**受け入れ条件:**
- 直近一覧から対象の支出を削除できる(スワイプ or 長押しメニュー)
- 削除後、一覧から即時消える
- 削除は取り消せない(確認ダイアログを表示する)

### US-03: 直近支出一覧

**As a** 支出を振り返りたい個人ユーザー  
**I want to** ホーム画面に直近の支出を一覧表示してほしい  
**So that** 最近の出費をひと目で把握できる

**受け入れ条件:**
- ホーム画面に最大20件の直近支出を日付降順で表示する
- 各行に「金額・カテゴリ・日付・メモ(先頭20文字)」を表示する
- 支出がゼロ件の場合は空状態メッセージを表示する

### US-04: 月次集計

**As a** 出費傾向を把握したい個人ユーザー  
**I want to** 月ごとの支出合計とカテゴリ別内訳をグラフで見たい  
**So that** どのカテゴリに使いすぎているかを視覚的に確認できる

**受け入れ条件:**
- 月ごとの支出合計を表示する
- カテゴリ別内訳を円グラフまたは棒グラフで表示する
- 表示月を前後に切り替えられる
- 支出がゼロ件の月は「データなし」状態を表示する

### US-05: 予算設定

**As a** 使いすぎを防ぎたい個人ユーザー  
**I want to** 月の予算金額を設定したい  
**So that** 月の支出が予算内に収まっているか管理できる

**受け入れ条件:**
- 月単位で予算金額(正整数・円単位)を設定できる
- 既存の予算は上書きできる
- 0円以下の予算は登録できない

### US-06: 予算超過警告

**As a** 使いすぎに気づきたい個人ユーザー  
**I want to** 月支出が予算の80%を超えたらホームに警告バナーを表示してほしい  
**So that** 使いすぎに早期に気づける

**受け入れ条件:**
- 月支出合計が予算の80%以上になったとき、ホーム画面に警告バナーを表示する
- 100%超過時は「予算超過」表示に変わる
- 予算未設定のときは警告バナーを表示しない

## Q&A ログ

### Q-01: カテゴリは固定リストにするか、ユーザーが自由に追加できるようにするか?

**質問背景:** 自由カテゴリは柔軟だが、集計ロジックが複雑になる。固定なら予測可能。  
**回答(D-01):** → 固定6カテゴリ(食費/交通/日用品/娯楽/医療/その他)に決定。MVP では固定が十分かつシンプル。カテゴリ追加は v0.1 以降に先送り。

### Q-02: 削除した支出は取り消せるようにするか(ソフトデリート)?

**質問背景:** ソフトデリートは安全だが、UI と Storage が複雑になる。  
**回答(D-02):** → ハードデリートのみ。確認ダイアログで誤操作を防ぐ。ゴミ箱機能は v0.1 以降に先送り。

### Q-03: 月次レポートのグラフ種別は円グラフか棒グラフか?

**質問背景:** 円グラフはカテゴリ構成比に強い。棒グラフは時系列比較に強い。  
**回答(D-03):** → カテゴリ別内訳は円グラフ。月別トレンドは棒グラフ。両方 SCR-03 に配置する。

## AI 独自決定

| ID | 決定内容 | 根拠 |
|---|---|---|
| D-01 | カテゴリを固定6種に決定 | MVP シンプル化。拡張容易。 |
| D-02 | 削除はハードデリート+確認ダイアログ | ストレージ複雑化を回避。 |
| D-03 | レポートに円グラフ+棒グラフ両方を配置 | カテゴリ構成比と月次比較、両方の情報ニーズに対応。 |
| D-04 | 予算は月単位で1件のみ(上書き方式) | 週次・カテゴリ別予算は複雑すぎる。最小有用単位が月次。 |

## 棄却案

| 案 | 棄却理由 |
|---|---|
| クラウド同期 | マルチデバイス要件なし。サーバー構築コストが MVP に見合わない。 |
| 繰り返し支出の自動登録 | ユースケースが少数。ドメインロジックが複雑化する。 |
| 自由カテゴリ追加 | D-01 参照。 |
| 週次予算 | D-04 参照。 |
| タグ付け | カテゴリと重複する機能。MVP スコープ外。 |
