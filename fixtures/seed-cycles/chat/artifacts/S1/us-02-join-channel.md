# US-02: チャンネル一覧を閲覧し参加する

## メタ
- 親: 要件一覧
- 対応 S2 画面: [SCR-01 メインレイアウト(サイドバー)](../S2/index.md)
- ステータス: 確定

## 3 視点

### なぜするか (Why)
チームに新しく加わったメンバーや、既存メンバーが新しいトピックを追いたいとき、どんなチャンネルが存在するかを一覧で確認できないと参加の入口がない。一覧から参加することで、全チャンネルを把握しながら必要なものに参加できる。

### UX へのインパクト
全チャンネルをサイドバーと別の「チャンネルブラウザ」画面で確認できる。参加済みのチャンネルはサイドバーに常に表示され、未参加のチャンネルはブラウザでボタン1つで参加できる。参加後はサイドバーに即時追加される。

### 受け入れ条件 (AC)
- サイドバーに参加済みチャンネルの一覧が表示される(チャンネル名 + 未読件数バッジ)
- 「チャンネルを追加」ボタンからチャンネルブラウザモーダルが開く
- モーダルには未参加のチャンネルが一覧表示される(チャンネル名 + 説明 + メンバー数)
- 「参加する」ボタンを押すと即座にサイドバーへ追加され、そのチャンネルへ遷移する
- 参加済みチャンネルはブラウザに「参加中」と表示され「参加する」ボタンが非活性になる

## この US 固有の AI が独自に決めたこと と 理由

### D-01 — チャンネルブラウザをモーダルで実装する
- **理由**: フルページ遷移にするとサイドバーコンテキストが失われる。モーダルならチャンネル一覧を見ながら操作でき、参加後の遷移がスムーズ。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし
