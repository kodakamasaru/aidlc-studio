## 3 視点

### なぜするか (Why)
チームが新しいトピックを始めるとき、既存のチャンネルに流し込むと話題が混在して読みにくくなる。専用チャンネルを手軽に作れることで、情報をトピック単位で整理できる。メンバーが自律的にチャンネルを作れることがチームの自己組織化を支える。

### UX へのインパクト
チャンネル名と説明を入力してボタンを押すだけで作成できる。作成直後にそのチャンネルへ自動遷移し、すぐにメッセージを投稿できる状態になる。他のメンバーは一覧に新チャンネルが即座に表示され、参加できる。

### 受け入れ条件 (AC)
- チャンネル名は 1〜50 文字の範囲で必須入力
- チャンネル名に使える文字は英数字・日本語・ハイフン・アンダースコア。スペースは不可
- 同じ名前のチャンネルが既に存在する場合、エラーメッセージ「そのチャンネル名はすでに使われています」が表示される
- 作成成功後、作成したチャンネルへ遷移し、サイドバーの一覧に新チャンネルが表示される
- 作成者は自動的にそのチャンネルのメンバーになる

## この US 固有の AI が独自に決めたこと と 理由

### D-01 — チャンネル名のスペース禁止
- **理由**: スペース入りチャンネル名は URL エンコードが複雑になり、コマンドラインや API 呼び出し時に扱いにくい。Slack 等の先行サービスでもスペース禁止が標準。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし
