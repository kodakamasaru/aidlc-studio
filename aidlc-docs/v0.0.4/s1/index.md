## 全体方針(グルーピング・優先度など)

主軸 = **「S1 を web で」live 会話ループ**。6 US は 3 つの塊に分かれる:

1. **live 成果物を本物に(基盤)**: US-01(前段文脈)/ US-02(md 描画)。これが無いと会話の前に「読める・連鎖した成果物」が出ない。
2. **会話を成立させる(主軸)**: US-03(質問が質問として出る)→ US-04(答えると resume)→ US-05(同一画面スレッド)。この 3 つで「AI が聞く→答える→進む」が web で回る。
3. **入口を会話に寄せる**: US-06(設定を一括ヒアリング)。個別フォームを廃し、設定も会話で埋める。

依存: US-03 → US-04 → US-05(質問の経路 → 継続 → UI)。US-06 は US-05 のスレッド基盤に乗る。US-01/02 は独立で先行可能。

優先度(推定): US-03/04 が主軸の心臓(無いと会話が成立しない)。US-01/02 は floor。US-05/06 は UX 完成。最終の版内優先は S5(作業単位)で確定。
