## 責務 (1〜2 行)
同一画面に QA を時系列で積み、画面遷移なしで連続回答できる対話ビュー(SCR-02)を作る。質問を S2 の 4 部テンプレ(質問/背景=折りたたみ/選択肢=★おすすめ+理由/自由入力)で描画し、返信を `aidlc-answers` にシリアライズして resume へ送る。設定ヒアリング(US-06)も同じ器に積み、設定の全文確認・修正面(SCR-04)を提供する(個別フォーム廃止)。既存 `AnswerView`(1 問 1 画面)/ `StepConfigPage`(フォーム)を置換。

## 外部依存
- [Unit-01](./unit-01-wire-contract.md) の wire 型 + `serializeAnswers`(質問描画と返信シリアライズの契約)。
- 実行時(S8 統合 / build を縛らない): Unit-03 の `QuestionRaised` を描画、返信が Unit-04 の resume を起動。並行 build 中は scripted アダプタ(Unit-04 が turn パリティ提供)に対して作る。
- 触る既存箇所: [web/src/features/inbox/](../../../web/src/features/inbox/)(`AnswerView.tsx` 置換)/ settings(`StepConfigPage` 置換)/ 既存 `QuestionPage`・`kind-meta.ts`・`question-summary.ts` の再利用。

## I/F 定義 (この Unit が公開する契約 = 画面の入出力)
| 操作 | 入力 | 出力 | エラー |
|------|------|------|--------|
| スレッド描画 | run の `Question[]`(時系列) | 4 部テンプレ AI バブル + 過去 QA を遡れるスレッド(見やすさ①/全文確認②) | 取得失敗は可視化(空表示で黙らない) |
| 連続回答 | 選択 + 自由補足(N 問一括 = バッチ) | `aidlc-answers` を resume へ POST(N 問→N 答→1 resume / S2 SCR-02) | ★おすすめ未選択でも自由入力で送信可。送信失敗は再試行導線 |
| 設定確認・修正(SCR-04) | 全ステップ × スコープ(グローバル既定/このサイクル) | 読み返し + 「会話で直す」導線(SCR-02 へ) | — |

- 状態表現(S2→S3 引き継ぎ): スレッドの「実行中(resume)」「stall + retry」「完了で離脱」。末尾追記のポーリング/更新。
