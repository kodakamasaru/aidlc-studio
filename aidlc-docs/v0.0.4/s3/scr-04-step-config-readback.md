## a11y
- **スコープは入口(entry context)で決まる / 対等タブ切替ではない**: グローバル設定(`/settings/steps`)を直接開いた場合は **既定のみ**(特定サイクルが無いので「このサイクル」タブは出さない)。サイクルから開いた場合は **そのサイクル**(既定を継承 + 範囲バッジ)。サイクル画面には「既定を編集 →」リンク(`<a>`)を置き、グローバルへ**明示的に画面遷移**する(silent なタブ切替で文脈を飛ばさない)。スコープの別は `<h1>` / 文脈タグ(全サイクル共通の既定 / サイクル vX·既定を継承)で読み上げる。
- **全ステップ一覧の読み上げ構造**: 実在ステップを行とする `<table>`(列: ステップ / 設定 / 範囲)。**ステップは番号(S1〜S12)でなく名前**(要件ヒアリング / UIデザイン / …)で表示・読み上げる(US-07 D-02 / 可変ステップで破綻させない)。スクリーンリーダーが「要件ヒアリング、成果物: 要件一覧、…、範囲: 既定」の順に読めるようにする。範囲バッジ(既定 / このサイクルで調整)はテキストで明示(色のみに依存しない)。実在しないステップの行・「省略」表記は出さない。
- **入力フォームは存在しない**: US-06 AC① に従い、このページに直接編集フォームは置かない。読み取り専用表示 + 「会話で直す」ボタンのみ。
- **pre-us 状態**: 要件(US)決定前はサイクル最適化がロック。`🔒` + テキストで「要件が決まると調整できます」を明示し、「会話で直す」ボタンは `aria-disabled="true"`。
- **loading 状態**: スケルトンを表示。`aria-busy="true"` をコンテナに付与する。

## pointer / keyboard 操作
- **「会話で直す(再ヒアリング)」**: クリック / Enter で SCR-02 スレッドへ遷移し、設定を会話で修正する。
- **empty 状態の「ヒアリングを始める」**: クリック / Enter で SCR-02 スレッドへ遷移し、初回ヒアリングを開始する。
- **出所リンク**: クリック / Enter で対応するスレッドの該当箇所へ遷移(またはハイライト)。Tab キーで到達できる。フォーカスリング(`outline: 2px solid var(--color-accent)`)を表示する。
- **ページ全体**: 入力フォームがないためフォームナビゲーション(F6 / H キー)は不要。ランドマークを適切に設置し、スクリーンリーダーのセクションジャンプを有効にする。

## motion
HTML の transition は参考にされない。motion 意図はここに文字で書く。

- **loading→default**: スケルトンから実コンテンツへの切り替えは 150ms `opacity` fade。
- **その他のブロック**: 静的。設定一覧はアニメーションなしで表示する。
- **reduced-motion**: loading fade も 0ms にフォールバックする。

## この画面固有の AI が独自に決めたこと と 理由
### D-03 — 全ステップ表示 + スコープは「入口で決まる」(対等タブ切替にしない)+ 範囲バッジに再設計(2026-06-13)
- **理由**: ユーザー指摘①「設定はサイクル全体に対してで、1 ステップに対してではないのでは?」(US-06 D-02)。②「タブ切替制だが、サイクルからでなくグローバル設定を最初から開いたらどうなる?」。既存 StepConfigPage は全ステップ 1 画面のグローバル設定。本画面を S1〜S12 全行 + 範囲バッジ(既定継承 / このサイクルで調整)に再設計し、**スコープは入口で固定**: グローバル(`/settings/steps`)= 既定のみ(サイクルタブ無し)、サイクル = そのサイクル + 「既定を編集 →」リンク。対等タブ切替は廃止(グローバル入口で「このサイクル」タブが空になる矛盾を解消)。`global`(既定 / 単体)/ `default`(このサイクル / US 後)/ `pre-us`(US 前=最適化ロック)/ `loading` で視覚契約化。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)

### D-01 — 要件(US)決定前は「会話で直す」をロックし、決定後にサイクル最適化を解放(pre-us 状態)
- **理由**: US-06 D-02「サイクル単位の最適化は US 決定後」。US 決定前は全ステップが既定継承で動き、最適化の余地がない。pre-us 状態で 🔒 + 文言を出し、ボタンを `aria-disabled` にする。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)

### D-02 — 範囲バッジ(既定 / このサイクルで調整)を各行に表示 / ステップ名から原文(SCR-06)へドリルイン
- **理由**: 全ステップ一覧では行ごとに「既定継承か / このサイクルで上書きか」が一目で要る → 範囲バッジ(テキスト併記で色覚非依存)。さらに **各ステップ名をクリックで指示・全文(原文 = SCR-06)へドリルイン**(2026-06-13 ユーザー指摘「原文はどこから?」への導線)。要約一覧 + 原文ドリルインで品質基準②「全文確認」を満たす。ステップ名に `›` を添え hover で強調。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)
