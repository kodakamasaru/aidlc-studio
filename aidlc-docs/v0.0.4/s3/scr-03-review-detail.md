## a11y
- **md 描画の見出し階層**: Markdown の `##` / `###` は HTML の `<h2>` / `<h3>` として描画し、スクリーンリーダーのランドマーク・ドキュメントアウトラインとして機能させる。見出しをフラットな `<p>` や `<strong>` で代替することは禁止。
- **コードフェンス**: ` ``` ` で囲まれたブロックは `<pre><code>` で描画し、等幅フォント・改行保持を保証する。スクリーンリーダーはコードブロックを verbatim に読み上げる。
- **md 内リンク**: 描画された Markdown 内のリンクはフォーカス可能(`<a href>`)。Tab キーで到達できる。
- **missing-context マーカー**: US-01 の唯一の視覚フットプリント。`role="alert"` 相当(assertive)で実装し、`⚠` 記号 + テキスト(例:「前サイクルの成果物が見つかりません — コンテキストが不完全な状態で実行されています」)を表示する。色のみに依存しない(stalled カラー + 記号 + テキストの三重符号化)。
- **アクションボタン**: 「承認して次 Phase へ」「差し戻し」はそれぞれ意味のある `aria-label` を持つ `<button>`。フォーカス可能。無効化状態は `aria-disabled="true"` で表現し、理由をツールチップまたは `aria-describedby` で補足する。
- **複数画像のギャラリー(D-03)**: 各サムネイルは `<figure>` + `<figcaption>`(画面名 + 状態)。`alt` に「{画面} {状態} のスクリーンショット」を入れる。グリッドは Tab で順に到達、Enter で拡大(lightbox)。lightbox は `role="dialog"` + `aria-modal="true"`、開いたら閉じるボタンにフォーカス、Esc で閉じ、‹ › は前後画像、`{n} / {total}` を読み上げる。色覚非依存(キャプションのテキストで画面/状態を明示)。

## pointer / keyboard 操作
- **「承認して次 Phase へ」**: クリック / Enter で Phase 完了を送信しボードへ戻る。確認ダイアログは持たない(v0 スコープ)。
- **「差し戻し」**: クリック / Enter で SCR-02 スレッドへ遷移し、差し戻し理由を会話で入力する。
- **md 内スクロール**: md 描画エリアはオーバーフロースクロール。コンテンツが長い場合も外側レイアウトはシフトしない。
- **md 内リンク**: hover でアンダーライン + cursor pointer。フォーカスリング(`outline: 2px solid var(--color-accent)`)を表示する。

## motion
HTML の transition は参考にされない。motion 意図はここに文字で書く。

- **loading→default**: スケルトン(プレースホルダーブロック)から実コンテンツへの切り替えは 150ms `opacity` fade。レイアウトシフトが起きないよう、スケルトンと実コンテンツは同一の高さ枠を占有する。
- **その他のブロック**: 静的。md 描画ブロックにはアニメーションを付けない(テキスト多量でパフォーマンスに影響するため)。
- **reduced-motion**: loading fade も 0ms にフォールバックする。

## この画面固有の AI が独自に決めたこと と 理由
### D-01 — missing-context マーカーを `role="alert"` 相当(assertive)に決定
- **理由**: 成果物の文脈が欠損したまま表示されることはユーザーへの重要な警告であり、`aria-live="polite"` では読み上げタイミングが遅れる可能性がある。SCR-02 の stall と同様に assertive を採用した。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)

### D-02 — 「差し戻し」後の遷移先を SCR-02 スレッドと明示
- **理由**: 差し戻し理由は会話スレッドで自然言語として伝えるのが US-04 の流れに整合する。専用フォームモーダルを追加するのは YAGNI。S2 D 決定に明示がない場合はこの D-02 が仮決定となりユーザー承認待ち。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)

### D-03 — 画像が複数のとき screenshot ブロックをキャプション付きギャラリーグリッド + クリック拡大(lightbox)で描く(2026-06-13 ユーザー指摘)
- **理由**: データモデルは screenshot ブロック = `{src, caption}` 1 枚単位ゆえ、複数画像 = 複数ブロック。素朴に全画面幅で縦積みすると S8/S9 の verify-ui 多数(本リポ S3 自身で 21 枚)で延々スクロールになり「人間が視覚で承認」しづらい。連続する screenshot ブロックを **キャプション(画面名+状態)付きのレスポンシブグリッド**にまとめ、**サムネをクリックで拡大(lightbox / ‹ › 前後 / n/total)** して各証拠を判読可能にする。`default`(2 枚)/ `gallery`(8 枚)/ `enlarged`(拡大)で視覚契約化。1 枚のときは従来どおり単一表示でよい(グリッドは複数時)。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)
