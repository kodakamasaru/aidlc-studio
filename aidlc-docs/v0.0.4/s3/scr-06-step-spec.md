## 目的
「ステップ設定の原文を見たいときどこから?」への答え。SCR-04(設定の要約一覧)から **ステップ名をクリックでドリルイン**し、そのステップの **① 画面から設定できる契約の全項目**(検証の観点の全文 / 成果物 / 人の確認 / 行き詰まり時)と **② AI が受け取る指示の本文(全文 = 原文)** を読み取り表示する。要約でなく全文を出すことで品質基準②「見ようと思えば全文確認できる」を満たす。確認のみ(編集は「ステップ設定」で会話)。

## native 固有挙動
- 本サイクルは web デスクトップのボード UI。native/モバイル固有挙動(safe area / status bar / swipe back 等)は対象外。

## a11y
- **読み上げ構造**: 契約の全項目は `<dl>` / `<dt>` / `<dd>`。AI への指示本文は `<pre>`(等幅・改行保持)で verbatim 読み上げ。見出し「設定の全項目」「AI への指示(全文)」は実 `<h2>` でランドマーク化。
- **パンくず**: 「‹ ステップ設定」はフォーカス可能な戻りリンク。
- **編集導線**: 「ステップ設定で編集 →」は本画面が読み取り専用であることを明示しつつ、編集面(SCR-04)へ誘導。
- **no-instruction**: 指示本文が無いステップは「未登録」を意味のあるテキストで示す(空白にしない)。
- ステップ名(原文の対象)は番号でなく**名前**(US-07 D-02 と整合)。

## pointer / keyboard 操作
- **入口**: SCR-04 の各ステップ名(`›` 付き / hover で強調)をクリック / Enter → 本画面。
- **編集導線**: 「ステップ設定で編集 →」で SCR-04 に戻り、会話で直す。
- **本画面は読み取り専用**: 入力フォームは無い(全文の確認のみ)。

## motion
HTML の transition は参考にされない。motion 意図はここに文字で書く。
- loading→default は 150ms `opacity` fade。本文(原文)は静的表示。
- reduced-motion で fade は 0ms。

## この画面固有の 質疑応答ログ
- **原文の入口(2026-06-13 ユーザー指摘)**: 「ステップ設定の原文はどこから?」→ SCR-04 の要約一覧では足りず、ステップ名からドリルインする本画面(指示・全文)が入口。既存 StepSpecPage が実体で、SCR-04 にあった「全文を見る」導線が再設計で抜けていたのを補う。
- **原文の範囲**: ① 画面設定できる契約の全項目 ② AI が受け取る指示本文(スキル本文)。本文の編集はここではしない(編集は会話 / SCR-04)。

## この画面固有の AI が独自に決めたこと と 理由
### D-01 — 原文を「契約の全項目 + AI 指示本文」の 2 ブロックで出す(StepSpecPage 踏襲)
- **理由**: 「原文」= 人が設定できる契約の全項目 と、AI が実際に受け取る指示本文の両方。要約(SCR-04)では②が見えないため、ここで両方を全文表示する。既存 StepSpecPage の構成を踏襲し最小差分。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)

### D-02 — 本画面は読み取り専用、編集は SCR-04(会話)へ集約
- **理由**: US-06 の「修正は会話一本化」。原文画面で直接編集を許すと会話一本化の契約と矛盾。確認と編集を分離。
- **判断**(ユーザー記入): **承認**(2026-06-13 確定)
