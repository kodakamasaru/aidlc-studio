# SCR-05: サイクル進捗(PhaseGroup 束ね)

## メタ
- 親: [s2/index.md](./index.md)
- 対応 US: [US-07](../s1/us-07-variable-step-progress.md)
- ステータス: 確定(2026-06-13 / S3 進行中に追加)

## 目的
サイクル画面の進捗表示を、可変ステップ数でも崩れず一目で読める形にする。既存 [PhasePipeline.tsx](../../../web/src/features/cycle-detail/PhasePipeline.tsx)(S1〜S12 を横一列 12 ノード)を置換し、**12 ステップを 5 PhaseGroup(Discovery / Design / Build / Validation / Improvement)に束ねた帯**にする。ステップ数が増減しても帯が吸収し、横溢れ/窮屈を防ぐ(US-07)。

## 主要要素
- **工程グループ帯(5 本)**: 各帯にラベル(要件 / 設計 / 実装 / 検証 / 改善)+ 内包ステップの**名前**チップ列。
  - 帯の状態: 完了(全ステップ done)/ 進行中(current を含む)/ 未着手。帯間コネクタは done 区間で緑。
- **ステップは「名前」で表示(番号なし / US-07 D-02)**: 固定番号(S1〜S12)は欠けたとき欠番か振り直しで破綻するため使わない。名前(要件ヒアリング / UIデザイン / …)で並べる。
- **ステップチップ**: 各ステップを色 + 記号で二重符号化 — 完了(✓ 緑)/ 進行中(● 青)/ 行き詰まり(! オレンジ)/ 未着手(○)/ 手戻り再入(↩ オレンジ)。
- **現在地の強調**: 現在の工程グループ帯と現在ステップを強調(リング/塗り)。
- **可変対応**: **実在ステップのみ描く**(`cycle.phases` 駆動 / US-07 Q-02)。あるステップが無いサイクルはその帯のチップが減るだけで帯は保たれる。「省略」ラベルは出さない(省略の記録はデータに無い)。
- **凡例**: 「● 青=進行中 / ✓ 緑=完了 / ○ 未着手」+ 手戻りがあれば ↩ 凡例。

## US-07 AC の充足
| AC | 充足点 |
|----|-------|
| 可変ステップでも破綻せず一目で現在地 | 工程グループ帯がステップ増減を吸収、現在帯/現在ステップを強調 |
| 番号でなく名前で表示 | 各ステップを名前チップで表示(欠番/振り直し/省略ラベルが発生しない) |
| 実在ステップのみ描く | `cycle.phases` 駆動。不在ステップは参照しない |
| 状態を色+記号で二重符号化 | ✓ 完了 / ● 進行中 / ! 行き詰まり / ○ 未着手 / ↩ 手戻り |
| 現在の工程グループ・ステップ強調 | 現在帯のリング、現在チップの塗り |
| 完了/未着手 工程グループの区別 | 帯ヘッダの ✓/●/○ + コネクタの緑塗り |
