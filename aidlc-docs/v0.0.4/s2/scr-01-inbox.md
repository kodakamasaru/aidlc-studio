# SCR-01: 対応待ち一覧(Inbox)

## メタ
- 親: [s2/index.md](./index.md)
- 対応 US: [US-03](../s1/us-03-output-question-routing.md)
- ステータス: 確定(2026-06-13)

## 目的
AI → 人間の依頼(質問 / できあがりの確認)が時系列で立つ入口一覧。live run が質問を含むとき、それが「レビュー(◎)」ではなく「質問(?)」として一目で区別できる(US-03 の魂)。

## 主要要素
- カード行: 種別アイコン + 平易ラベル(? 質問 / ◎ できあがりの確認 / 実機確認 …)、タイトル、メタ(ステップ / 時刻)、アクション動詞(回答する / 確認する)。
- 種別の区別: `question`(?, 回答する)と `visual_review`(◎, 確認する)を視覚的に分離(既存 kind-meta を踏襲)。
- カードを開く → 種別で遷移先が変わる: 質問 → SCR-02 スレッド / できあがりの確認 → SCR-03 レビュー詳細。


## US-03 に対する S2 の論点
- 既存 Inbox は既に `question` / `visual_review` を区別表示できる(kind-meta.ts)。S2 で新規に必要なのは **live run の出力が `question` 経路に乗る**こと(backend)で、Inbox の見た目は据え置きでよい。
- 1 run が複数質問を含むとき(US-03 AC)、複数カード or スレッド内で順次提示。本画面では「複数カードが立つ」を既定とし、開くと同一スレッド(SCR-02)に合流。
