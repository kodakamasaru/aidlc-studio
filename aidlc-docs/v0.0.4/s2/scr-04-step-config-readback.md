# SCR-04: ステップ設定の確認・修正

## メタ
- 親: [s2/index.md](./index.md)
- 対応 US: [US-06](../s1/us-06-bulk-hearing.md)
- ステータス: 確定(2026-06-13)

## 目的
AI ヒアリング(SCR-02)で埋まったステップ設定を**全文で確認**し、おかしければ**手軽に直す**ための面。既存 `StepConfigPage`(個別フォーム欄の集合)を置換し、**個別フォーム欄を UI から無くす**(US-06 AC ①)。

## 主要要素
- 設定サマリ(読み取り表示): 検証の観点 / 成果物の種類 / 人の確認モード / 行き詰まり時の対応 / 最大やり直し回数 を、フォーム欄でなく**読みやすい一覧**で表示。
- 各値に「どのヒアリングで埋まったか」の出所(該当スレッドへのリンク)。
- 修正導線:「会話で直す(再ヒアリング)」→ SCR-02 スレッドに戻る(品質基準③ 手軽に根治)。
- 個別入力フォーム欄は出さない(廃止)。

## US-06 AC の充足
| AC | 充足点 |
|----|-------|
| 個別フォームでなくヒアリングで埋まる | 入力は SCR-02、本画面は結果の読み返し |
| 埋まった設定の全文確認(②) | 全項目を一覧表示、出所スレッドへ辿れる |
| 手軽に修正(③) | 「会話で直す」で SCR-02 へ。直接編集は持たない(会話一本化) |
| 個別フォーム欄が消える(①) | 本画面に入力フォームは無い |
| US-05 のスレッド基盤に乗る | 修正・ヒアリングは SCR-02 を共用 |

- **回答**(ユーザー記入):

- **確定**(AI 記入):

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — 修正は「会話で直す」に一本化し、個別値の直接編集フォームは持たない
- **理由**: US-06 の主眼は「個別設定欄はいらない / 会話で埋める」。直接編集フォームを残すと廃止対象のフォームが温存され ① に反する。修正も会話で根治(③)。
- **判断**(ユーザー記入): **承認**(2026-06-13)
- **上書き内容**(上書き時のみ):

### D-02 — 純粋に技術的な既定値(人間が選ぶ性質でない設定)は本画面に出さず AI 既定のまま
- **理由**: US-06 Q-01 確定「人間が値を選ぶ性質の設定をヒアリング化、技術既定は AI 既定」。確認面に出すのは人間が判断する値のみ。対象の最終線引きは S5 の棚卸しで確定。
- **判断**(ユーザー記入): **承認**(2026-06-13)
