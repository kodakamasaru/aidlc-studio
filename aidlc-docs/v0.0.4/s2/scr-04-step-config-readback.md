# SCR-04: ステップ設定の確認・修正

## メタ
- 親: [s2/index.md](./index.md)
- 対応 US: [US-06](../s1/us-06-bulk-hearing.md)
- ステータス: 確定(2026-06-13)

## 目的
AI ヒアリング(SCR-02)で埋まった**サイクルの全ステップ(S1〜S12)の設定**を**全文で確認**し、おかしければ**会話で手軽に直す**ための面。既存 `StepConfigPage`(全ステップ × 個別フォーム欄の集合)を置換し、**個別フォーム欄を UI から無くす**(US-06 AC ①)。**1 ステップ単位ではなく全ステップを束ねる**(US-06 D-02)。

## 主要要素
- **スコープは入口で決まる(対等タブ切替ではない)**: グローバル設定(`/settings/steps`)を直接開く = 既定のみ(「このサイクル」タブは出さない / 特定サイクルが無いため)。サイクルから開く = そのサイクル(既定を継承 + 範囲バッジ)+「既定を編集 →」リンクでグローバルへ明示遷移。既定は今後のサイクルの既定になる(作成済みは固定)。このサイクルは要件(US)決定後の最適化。
- **全ステップ一覧(読み取り表示)**: 実在ステップを**名前**で行に並べ(番号 S1〜S12 は出さない / US-07 D-02)、各行に主要設定(成果物の種類 / 人の確認 / 行き詰まり時 等)を表示。フォーム欄でなく読みやすい一覧。
- **範囲バッジ**: 各ステップ行に「既定」(継承)/「このサイクルで調整」(上書き)を表示。空欄は既定を継承。
- **原文への導線(品質基準②)**: 各ステップ名をクリックすると、そのステップの**指示・全文(原文)= SCR-06**へドリルイン(契約の全項目 + AI への指示本文)。要約一覧では足りない「全文確認」をここで満たす。
- 修正導線:「会話で直す(再ヒアリング)」→ SCR-02 の全ステップ設定ヒアリングに戻る(品質基準③)。
- 個別入力フォーム欄は出さない(廃止)。

## US-06 AC の充足
| AC | 充足点 |
|----|-------|
| 個別フォームでなくヒアリングで埋まる | 入力は SCR-02、本画面は結果の読み返し |
| 設定の単位は全ステップ | S1〜S12 を 1 画面で一覧(1 ステップ単位でない) |
| 2 層モデル(既定 / サイクル最適化) | 入口で決まるスコープ(グローバル=既定のみ / サイクル=このサイクル + 既定を編集リンク)+ 範囲バッジ(既定継承 / このサイクルで調整) |
| 埋まった設定の全文確認(②) | 全ステップ・全項目を一覧表示 |
| 手軽に修正(③) | 「会話で直す」で SCR-02 へ。直接編集は持たない(会話一本化) |
| 個別フォーム欄が消える(①) | 本画面に入力フォームは無い |
| US-05 のスレッド基盤に乗る | 修正・ヒアリングは SCR-02 を共用 |

- **回答**(ユーザー記入):

- **確定**(AI 記入):

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — 修正は「会話で直す」に一本化し、個別値の直接編集フォームは持たない
- **理由**: US-06 の主眼は「個別設定欄はいらない / 会話で埋める」。直接編集フォームを残すと廃止対象のフォームが温存され ① に反する。修正も会話で根治(③)。
- **判断**(ユーザー記入): **承認**(2026-06-13)
- **上書き内容**(上書き時のみ):

### D-02 — 純粋に技術的な既定値(人間が選ぶ性質でない設定)は本画面に出さず AI 既定のまま
- **理由**: US-06 Q-01 確定「人間が値を選ぶ性質の設定をヒアリング化、技術既定は AI 既定」。確認面に出すのは人間が判断する値のみ。対象の最終線引きは S5 の棚卸しで確定。
- **判断**(ユーザー記入): **承認**(2026-06-13)
- **上書き内容**(上書き時のみ):

### D-03 — 全ステップ × スコープ切替(グローバル既定 / このサイクル)に再設計(2026-06-13 / 旧 1 ステップ scope を撤回)
- **理由**: ユーザー指摘「設定はサイクル全体に対してで、1 ステップに対してではないのでは?」。初版は 1 ステップ scope(「ステップ設定 — S1 要件ヒアリング」)だったが、既存 StepConfigPage は全ステップを 1 画面に持つグローバル設定で誤り。US-06 D-02 の 2 層モデルに合わせ、本画面を S1〜S12 全行 + スコープ切替(既定 / このサイクル)+ 範囲バッジに再設計。サイクル最適化は要件(US)決定後。
