## このサイクルの S2 の主眼

主軸 = **「S1 を web で」live 会話ループ**(S1 全体方針)。S1 で構造論点(対話ビュー統合 / turn ベース / 構造化 emit / 双方向フォーマット化)は確定済み。S2 の仕事は **その構造を絵に起こし、対話メッセージのテンプレ形(D-04 の具体化)を決める**こと。

既存 UI からの主な置換:
- `AnswerView`(1 問 1 画面 → 回答後に離脱)→ **SCR-02 会話スレッド**(同一画面・時系列・連続回答)
- `StepConfigPage`(個別フォーム欄)→ **SCR-04 設定の確認・修正**(フォーム廃止 / 読み返し + 会話で直す)
- `ReviewDetail` の summary body(プレーンテキスト)→ **SCR-03** で md 描画

## 対話メッセージのフォーマット(D-04 の具体化 / S2 確定対象)

S1 D-04「対話の双方向をフォーマット化する」を S2 で具体テンプレ化する。**表示・入力の形**を S2 で固め、wire/JSON スキーマの詳細は S4 に送る。

### AI → 人間 の質問テンプレ(SCR-02 で AI バブルに描画)
```
【質問 N】 <一文の問い>
【背景】   <なぜ聞くか。前段の文脈・この選択が後工程に効く理由。長文は折りたたみ>
【選択肢】 (任意。無ければ自由記入のみ)
  ① <ラベル> — <ヒント>            ★AI のおすすめ
  ② <ラベル> — <ヒント>
  ③ その他(自由に回答)
【回答形式】 単一選択 | 複数選択 | 自由記入
```

### 人間 → AI の返信テンプレ(SCR-02 の入力 → resume に渡す)
```
【回答 → 質問 N】
選択: ②                 (選択肢があるとき)
補足: <自由記入>          (任意 / 「その他」選択時は本文)
```

- **見やすさ①**: 既定は【質問】+【選択肢】を前面、【背景】は折りたたみ(展開で全文)。
- **全文確認②**: 折りたたみを開けば背景・過去 QA の全文をスレッドで遡れる。
- **機械処理**: AI 側は構造化マーカ(S1 Q-03 確定)で emit、live adapter が `QuestionRaised` に変換。人間の返信も上記テンプレで構造化し resume に渡す。

## 棄却した画面案

### R-01 — US-01 用に「前段文脈プレビュー画面」を新設する
- **棄却理由**: v0.0.4 scope 外(run ログ/可視化はリッチ可視化 backlog)。注入は backend 責務で、独立画面は scope を膨らませる。マーカーは合成 prompt 内 + レビューでの体感で足りる。
