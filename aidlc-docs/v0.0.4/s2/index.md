## このサイクルの S2 の主眼

主軸 = **「S1 を web で」live 会話ループ**(S1 全体方針)。S1 で構造論点(対話ビュー統合 / turn ベース / 構造化 emit / 双方向フォーマット化)は確定済み。S2 の仕事は **その構造を絵に起こし、対話メッセージのテンプレ形(D-04 の具体化)を決める**こと。

既存 UI からの主な置換:
- `AnswerView`(1 問 1 画面 → 回答後に離脱)→ **SCR-02 会話スレッド**(同一画面・時系列・連続回答 / 1 ターン複数質問のバッチ)

- `ReviewDetail` の summary body(プレーンテキスト)→ **SCR-03** で md 描画
- `PhasePipeline`(横一列 12 ノード)→ **SCR-05 サイクル進捗**(5 PhaseGroup 帯で可変ステップ吸収 / 2026-06-13 追加)

## 対話メッセージのフォーマット(D-04 の具体化 / S2 確定対象)

S1 D-04「対話の双方向をフォーマット化する」を S2 で具体テンプレ化する。**表示・入力の形**を S2 で固め、wire/JSON スキーマの詳細は S4 に送る。

### AI → 人間 の質問テンプレ(SCR-02 で AI バブルに描画)

**全質問共通の必須 3 要素(2026-06-13 確定 / D-04)**: ①どんな質問でも選択肢を出す ②必ず ★おすすめを 1 つ付け、おすすめ理由を書く ③自由入力欄を毎回常設(その他/補足)。= Claude の質問窓と同じ感覚(各選択肢にラベル+説明、おすすめ印+理由、自分で書く欄)。

```
【質問 N】 <一文の問い>
【背景】   <なぜ聞くか。前段の文脈・この選択が後工程に効く理由。既定は折りたたみ>
【選択肢】 (必ず提示。数値・自由系でも AI が候補を出す)
  ① <ラベル> — <説明>              ★おすすめ
       おすすめ理由: <なぜこれが良いか>
  ② <ラベル> — <説明>
  ③ <ラベル> — <説明>
【自由入力 / 補足】 (毎回常設。選択肢に無ければここに自由記入 / 補足もここへ)
  [ ............................. ]
【回答形式】 単一選択 | 複数選択(+ 常に自由入力可)
```

### 人間 → AI の返信テンプレ(SCR-02 の入力 → resume に渡す)
```
【回答 → 質問 N】
選択: ①                 (選択肢から)
補足: <自由入力>          (任意 / 自由入力欄の内容)
```

- **見やすさ①**: 既定は【質問】+【選択肢(★おすすめ+理由)】を前面、【背景】は折りたたみ(展開で全文)。
- **全文確認②**: 折りたたみを開けば背景・過去 QA の全文をスレッドで遡れる。
- **おすすめ理由**: ★おすすめの選択肢には必ず「おすすめ理由」を併記し、推す根拠を可視化(推すだけでも進めるサクサク感)。
- **機械処理**: AI 側は構造化マーカ(S1 Q-03 確定)で emit、live adapter が `QuestionRaised` に変換。人間の返信も上記テンプレで構造化し resume に渡す。選択肢・おすすめ・理由・自由入力の wire/JSON スキーマは S4。
