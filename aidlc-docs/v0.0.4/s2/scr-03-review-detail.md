# SCR-03: 成果物レビュー詳細

## メタ
- 親: [s2/index.md](./index.md)
- 対応 US: [US-02](../s1/us-02-markdown-render.md)(主) / [US-04](../s1/us-04-conversational-resume.md)(完了の着地点) / [US-01](../s1/us-01-prior-artifact-context.md)(可視マーカー footprint)
- ステータス: 確定(2026-06-13)

## 目的
できあがった成果物を web 上で正しい構造で読み、IDE を開かずに承認/差し戻しを判断する画面。既存 `ReviewDetail` を踏襲しつつ、summary block の body を **Markdown 描画**にする(US-02)。harness 原則③「人間がコードを見ずに承認できる成果物」の前提。

## 主要要素
- ヘッダ: パンくず + 種別バッジ(◎ できあがりの確認)+ タイトル + メタ。
- ブロックストリーム: 既存ブロック(ac-map / screenshot / risk / test / coverage / diff)はそのまま。
- **summary block の body を md 描画**(見出し / 箇条書き / 番号付き / コードブロック / 表 / リンク)。
- アクション: 「承認して次 Phase へ」/「差し戻し(backtrack モーダル)」。

## モック (ASCII)

```
+----------------------------------------------------------------+
| Inbox › cycle v0.0.5 › S2 レビュー        [承認] [差し戻し]     |
+----------------------------------------------------------------+
| ◎ できあがりの確認 — S2 画面モック                             |
+----------------------------------------------------------------+
|  ▌summary                                                      |
|   ## 画面構成                       ← md 見出しとして描画        |
|   - SCR-01 Inbox                    ← 箇条書き                  |
|   - SCR-02 会話スレッド                                         |
|   ```                                                          |
|   flowchart …                       ← コードフェンスは崩さず表示 |
|   ```                                                          |

|  ▌ac-map        AC1 ✓ / AC2 ✓ / AC3 ―                          |
|  ▌screenshot    [ 画像 ]                                       |
|  ▌risk          中: …                                          |
+----------------------------------------------------------------+
```

## US 別の充足
| US | この画面での充足点 |
|----|------------------|
| US-02 | summary body が md 構造で描画(安全サニタイズ / 外部 CDN 非依存 / 既存ブロック非破壊) |
| US-04 | live resume が「成果物完成」で着地したとき、ここに ◎ レビューとして出る |
| US-01 | 前段文脈が反映されているかを人間がここで体感。欠落時の「※ 前段文脈が見つかりません」マーカーが本文に現れうる |
