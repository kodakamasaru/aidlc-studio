- ステータス: 確定(2026-06-13)

## 目的
1 つのステップの会話(AI の質問・人間の回答・設定ヒアリング)が**同一画面に時系列(スレッド)で積み上がり、画面遷移なしで連続して答えられる**対話ビュー。既存 `AnswerView`(1 問 1 画面 → 離脱)を置換する、live 会話ループの心臓。

## 主要要素
- ヘッダ: ステップ名 + 状態(ヒアリング中 / 実行中(resume)/ 完了 / stall)。
- スレッド本体(時系列・上が古い):
  - **AI バブル**: index の質問テンプレ(【質問】+【背景(折りたたみ)】+【選択肢】+【回答形式】)。**1 ターンで確認したい質問を複数まとめて 1 つの AI バブルに提示する**(後述 D-04 / 2026-06-13 reconcile)。
  - **人間バブル**: 返信テンプレ(選択 + 補足)。**バッチへのまとめ回答は 1 つの人間バブルに集約**(質問 N 件 → 回答 N 件)。
- 入力(各質問にインライン): 質問ごとに選択肢ラジオ / 自由記入を直下に置き、バブル末尾の **「まとめて送信して再開」1 ボタン**で全回答を 1 回の resume に渡す(Cmd/Ctrl+Enter)。個別に 1 問ずつ送る画面下固定ドックは廃し、バッチ一括送信に統一(D-04)。
- 状態表示: 回答送信後「実行中…(AI が続きを考えています)」、次の質問が来たら**末尾に追記**(ポーリング/更新)。成果物が完成したら Inbox に ◎ が立ち、本スレッドは「完了」表示。
- stall(resume 失敗/timeout): スレッド内に stall 表示 + 「再試行」。黙って失わない(US-04 AC)。
- 全文遡り: 過去の QA は同画面でスクロールして全文確認できる(US-05 AC / 品質基準②)。

## 設定ヒアリングモード(US-06)での見え方
- 同じスレッド器を使う(index Q-01 確定)。ステップ着手で AI が「設定を埋めるための質問群」をまとめて投げ、人間は会話で答える。
- 個別フォーム欄は出さない(US-06 AC ①)。
- ヒアリングが一段落したら、埋まった設定の確認導線(SCR-04)を提示。

## 工程の再構成モード(US-08)での見え方
同じスレッド器を使う。**要件(S1)確定の直後に 1 回**、AI が「このサイクルの工程をこう組み直しました」という**再構成案**を 1 つの AI バブルで提示する。

- **再構成案 = 既定構成との差分表示**(D-05a): 各工程に `既定のまま` / `✕【削除】+理由` / `＋【新設】` / 並べ替え を付け、一目で何が変わったか分かる。**着手済み(要件=完了)は固定**(US-08 AC-5)。
- 各工程の**ルール本文(指示 md)は「▸ 各ルールを確認」で展開して読むだけ**(D-05b)。要件に合わせて作り直したもの(US-08 AC-2/3)。直したいときは会話で指示し、AI が案を更新して再提示。
- **修正は会話で**(個別フォーム/インライン編集なし / US-06 会話主義): 「○○を足して」「△△は要らない」「順番を入れ替えて」→ AI が再構成案を更新→再提示(承認まで往復)。独自工程の新設も会話で(US-08 AC-3)。
- **承認は全体まとめて 1 回**(D-05c): 「この構成で承認」で、その構成がこのサイクルの実工程列になり、進捗(SCR-05)・設定確認(SCR-04)に反映。
- 状態ヘッダ = 「工程の再構成(確認待ち)」。承認後は通常のステップ進行へ。

### 2 モード(起点が逆 / US-08 AC-7)
US-08 の再構成は **2 モード**。上記は**サイクル側(AI 起点)**。もう一方は**グローバル既定編集(人間起点)**で向きが逆(D-06)。

| | サイクル側(US あり) | グローバル既定(US なし) |
|---|---|---|
| 起点 | **AI**(S1 確定直後に自動で再構成案) | **人間**(SCR-04 global「会話で直す」を自分から開く) |
| 最初の発話 | AI が差分付き再構成案 | **AI が現在の既定構成を提示**(参照)+「どこを変えますか?」(D-e) |
| 入力 | US + 既定構成 + ヒアリング | 既定構成 + 人間の指示(US なし) |
| 差分基準 | 既定との差分を表示 | 既定そのものを編集ゆえ差分基準なし |
| 反映先 | このサイクルの実工程 | 今後作るサイクルの既定(作成済みは不変 / US-06 2層) |

グローバル既定モードでも、新設工程のルール md は **人間の説明から**生成(US でなく)。承認の往復・会話修正・ルールは読むだけ、は両モード共通(D-05)。

## US 別の充足
| US | この画面での充足点 |
|----|------------------|
| US-03 | AI の質問が AI バブルとして(レビューでなく)スレッドに立つ |
| US-04 | 回答送信 → live resume → 次の質問/完了が**末尾に追記**。stall は再試行可 |
| US-05 | 同一画面・時系列・連続回答 / 過去 QA 全文遡り / Cmd+Enter 送信 |
| US-06 | 設定ヒアリングも同じスレッドで会話的に実施(個別フォーム廃止) |
| US-08 | 要件確定後の**工程再構成案**を同じスレッドで差分提示→会話で修正→まとめて承認 |

- **回答**(ユーザー記入): (AI 裁量に委任 — 実装方式の論点)
- **確定**(AI 記入):

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — AI/人間を左右(または色)で分けたチャット型スレッドにする
- **理由**: 「誰の発話か」を一目で。時系列の連続性(US-05 主眼)を最優先し、装飾は最小(D-01 of US-05)。
- **判断**(ユーザー記入): **承認**(2026-06-13)
- **上書き内容**(上書き時のみ):

### D-02 — 入力域は画面下固定。選択肢ラジオ + 自由記入 textarea を併置、Cmd/Ctrl+Enter で送信
- **理由**: 既存 AnswerView の操作感(ラジオ + mono textarea + Cmd+Enter)を踏襲し連続回答を軽く(US-05 AC)。
- **判断**(ユーザー記入): **承認**(2026-06-13)
- **上書き内容**(2026-06-13 D-04 で更新): バッチ一括回答に伴い「画面下固定の単一ドック」は廃止。各質問にインライン入力 + バブル末尾の「まとめて送信」1 ボタンに置換。Cmd/Ctrl+Enter は踏襲。

### D-04 — 1 ターン = 複数質問のまとまり(バッチ)。まとめて提示・まとめて回答・1 回 resume(2026-06-13 reconcile)
- **理由**: ユーザー指摘「質問って複数まとめてくるもんじゃないの?」。S2 初版は AI バブルを 1 問ずつ(ドリップ)で描いていたが、ヒアリング節は「質問群をまとめて投げる」とあり**内部矛盾**していた。IDE の `/aidlc-sN`(Q-NN をまとめて md に吐く)に対し、ドリップは退化で round-trip 過多 → §I #1「IDE よりサクサク」/ §I #2「一括ヒアリング」/ S1 D-04「双方向フォーマット化」に反する。よって **1 ターンで AI が確認したい質問を複数まとめて提示し、人間は各問にインラインで答えて 1 回で送信 → 1 回 resume** に確定。turn ベース(S1 Q-02)と整合(1 turn が複数 Q を内包)。US-03(複数質問の経路)/ US-05(連続回答)/ US-06(設定ヒアリングも同じバッチ器)に反映。
- **判断**(ユーザー記入): **承認**(2026-06-13 / ユーザー指摘起点・バッチ見せ方「まとめて表示＋まとめて回答」を選択)
- **上書き内容**(上書き時のみ):

### D-05 — 工程の再構成モード(US-08): 差分提示 / ルールは読むだけ+会話修正 / まとめて1回承認
- **理由**: US-08 を同じスレッド器に乗せる。a)再構成案は**既定との差分(既定のまま/削除+理由/新設/並べ替え)**で見せ、何が変わったか一目に。b)各工程のルール本文は**読むだけ**(展開で全文)、直しは会話で指示→AI 再提示(US-06 会話主義 / 個別フォーム・インライン編集なし)。c)承認は**全体まとめて 1 回**(工程ごと個別承認はしない)。最小実装かつ会話主義に整合。
- **判断**(ユーザー記入): **承認**(2026-06-14 / D-a 差分表示・D-b 読むだけ+会話修正・D-c まとめて1回 = 「全部ok」)
- **上書き内容**(上書き時のみ): 

### D-06 — グローバル既定の編集は「人間起点」モード(AI が現既定を見せて assist / 差分基準なし)
- **理由**: ユーザー指摘「デフォルトをいじる時は人間側が起点になりそう」。US-08 AC-7 の 2 モードのうち、グローバル既定編集は US が無いため AI が S1 から再構成案を出せない。よって**人間が起点**: SCR-04 global「会話で直す」で人間が開始 → **AI はまず現在の既定構成を提示**(再構成案でなく参照)+「どこを変えますか?」(D-e)→ 人間が指示 → AI が変更案+新設工程のルール md(人間の説明から生成)を提示 → 承認の往復。差分基準は無い(既定そのものを編集)。承認で既定を更新(今後のサイクルの出発点 / 作成済みは不変)。サイクル側(AI 起点・US 駆動・差分表示)とは起点と入力が逆。
- **判断**(ユーザー記入): **承認**(2026-06-14 / D-d 人間起点+AI assist+差分基準なし・D-e AI が現既定提示で開始 = 「推奨通りで」)
- **上書き内容**(上書き時のみ):
