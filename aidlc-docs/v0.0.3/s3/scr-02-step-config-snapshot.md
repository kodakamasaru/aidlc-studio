## 視覚契約(この差分が S7/S8 への実装契約)

既存のステップ設定画面(v0.0.2)に **控えめな注記バナーを 1 つ追加するだけ**。フォーム・ステップカードなど他要素は不変。

### バナー仕様

| 項目 | 値 |
|------|-----|
| 文言(確定) | **「ⓘ ここでの編集は “これから作る” サイクルに反映されます。作成済みのサイクルは作成時点の構成に固定です。」** |
| 配置 | 画面説明文の直下、ステップカード群の上 |
| 見た目 | indigo 系の淡い面(背景 ~10% / 枠線 ~45% の indigo)・角丸・本文より小さい字。既存トークン(indigo / violet・4px base)の範囲内。**常時表示の控えめバナー**(閉じるボタン無し) |
| トーン | 警告ではなく案内。色は accent(indigo)で alarming にしない |

> 注: 文言に「(再現性のため)」は **入れない**(2026-06-12 Biz レビュー指示)。理由づけは UI に出さず、内部ドキュメント(S1/US-02)に残す。

- 同じ一文を **サイクル側のステップ構成ビュー**上部にも置く(「このサイクルのステップは作成時点で確定。以後デフォルトを変えてもこのサイクルには反映されません。」)。専用画面は作らない。

- コントラスト: 淡い面 + 本文色で WCAG AA を満たす明度を保つ。

### D-01 — accent(indigo)の淡い面で「案内」トーンにする(warning 色にしない)
- **理由**: スナップショット挙動は正常仕様であって警告ではない。amber/red にすると誤って「問題」に見える。indigo の淡面で落ち着いた案内にする。

## この画面固有の 棄却した案
### R-01 — ツールチップ(ⓘ ホバー)だけにする
- **棄却理由**: ホバーは気づかれない。スナップショット挙動は誤解されると「編集したのに反映されない」と見えるため、常時可視の一文で先回りする。
