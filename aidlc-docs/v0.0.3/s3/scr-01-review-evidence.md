## 視覚契約(この差分が S7/S8 への実装契約)

既存のレビュー詳細(v0.0.2)の中で、変わるのは **「実際に動いた証拠」ブロックの中身だけ**。周囲(ヘッダ / 対応状況テーブル / まとめ / 影響 / 操作の動画枠 / 承認・差し戻しボタン)は v0.0.2 のまま不変。

### 状態網羅

| 状態 | 見た目 | 契機 |
|------|--------|------|
| default(実画像あり) | 証拠枠に実スクリーンショットを **枠幅にフィット**(角丸・枠線は既存 screenshot-block 踏襲)。キャプション「実際に動いた画面」 | live verify-ui がスクショ取得成功 |
| failed(取得失敗) | 既存の placeholder 枠(破線・中央アイコン)を維持し、文言を **「スクリーンショット取得に失敗: {理由}(再取得できます)」** に。空表示にしない | live verify-ui がスクショ取得失敗 |

- **completeness テーブル**(「やりたかったことの 対応状況 2/2」)は描画不変。v0.0.3 では **live がデータの driver**になるが、見た目・コンポーネントは既存流用(US-04)。

## native 固有挙動
- なし(web プロダクト)。

## a11y
- 実画像 `<img>` に `alt="実際に動いた画面"`。装飾でなく証拠なので意味のある alt を付ける。
- failed 時の placeholder は `role="img"` + `aria-label` に失敗理由を含める(読み上げで「証拠が無い」と「取得に失敗」を区別できる)。
- 色コントラスト: 既存 review block の基準(WCAG AA)を踏襲。失敗文言は本文色で読める明度を保つ。

## gesture
- 実画像クリックで原寸モーダル表示(暫定 / 詳細は実装時に確定。最低限、枠フィット表示で承認判断はできる)。

## motion
- 既存 review block の stagger 入場(fade + slide-up, ~200–300ms ease-out)を踏襲。実画像も同じ入場に乗せる。
- 失敗 placeholder も同じ入場。点滅・スピナーの追加はしない(静的に理由を出す)。
