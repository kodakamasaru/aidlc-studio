## 3 観点

### なぜするか (Why)
人間は次に何を作るべきか思いつかないことがある。AI がビジョン(brief)と現状から「次の開発要求」を起案できれば、人間は発想ではなく取捨選択に集中でき、Backlog が枯れない。

### UX へのインパクト
ユーザーは「次に何かある?」と AI に問い、ビジョン整合の取れた Task 候補を提示してもらえる。気に入った候補を Backlog に採用するだけで、計画づくりの初動が軽くなる。

### 受け入れ条件 (AC)
- [v0.0.x] AI がビジョンと現状(既存 Backlog/成果物)を踏まえて Task 候補を起案する
- 各候補にタイトル・概要・起案根拠が添えられる
- ユーザーは候補を採用(Backlog 追加)または破棄できる
- 採用された候補は通常の Task と同様に扱える
- 提案はあくまで叩き台であり、人間が最終判断する
