## 3 観点

### なぜするか (Why)
開発要求(機能追加・改善・バグ修正など)を思いついた時点で取りこぼさず溜める場所が要る。Backlog は「次に AI-DLC を回す候補」の源泉であり、ここが空だと Cycle を組めない。

### UX へのインパクト
ユーザーは IDE を開かずサイト上で要求をタイトル+概要で登録でき、頭の中の TODO を即座に台帳化できる。後で Cycle に束ねるまでは「積んでおくだけ」で良い。

### 受け入れ条件 (AC)
- [v0.0.x] Backlog にタイトル(必須)と概要(任意)を入力して Task を 1 件追加できる
- 追加した Task は未割り当て状態で Backlog 一覧の末尾に出現する
- 各 Task に一意 ID と作成日時が自動付与される
- タイトル未入力では追加できず、エラーが表示される
- 対象リポジトリ(PJ)に紐づいた Backlog に登録される
