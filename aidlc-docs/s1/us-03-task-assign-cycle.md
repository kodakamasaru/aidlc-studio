## メタ

- ステータス: 確定

## 3 観点

### なぜするか (Why)
AI-DLC の実行単位は Cycle であり、どの Task をその Cycle で回すかは人間が決める中核判断。手動割り当てができないと、束ね方を人間が完全にコントロールできない。

### UX へのインパクト
ユーザーは Backlog から Task を選び、対象 Cycle に明示的に放り込める。束ねる意図(関連機能をまとめる等)を自分の手で表現できる。

### 受け入れ条件 (AC)
- [v0.0.x] Backlog の未割り当て Task を、指定した Cycle に割り当てられる
- 割り当て済み Task は Backlog 上で「割当済み(Cycle 名)」として識別できる
- 1 つの Cycle に複数 Task を割り当てられる
- 割り当てを解除して Task を Backlog へ戻せる
- 実行中(走行開始後)の Cycle への追加可否は制御される(走行前のみ追加可)
