## 3 観点

### なぜするか (Why)
何を先に回すかは状況で変わる。Backlog 内で優先順位を見える形で持てないと、次に束ねるべき Task の判断がブレる。優先順位は人間の判断の一次入力になる。

### UX へのインパクト
ユーザーはドラッグ等で Task を並べ替え、上から順に「やる順番」を直感的に表現できる。AI の割り当て提案(US-04)もこの順位を尊重できる。

### 受け入れ条件 (AC)
- [v0.0.x] Backlog 内の Task をドラッグ&ドロップで任意の位置に並べ替えられる
- 並べ替え後の順序が永続化され、再読み込み後も保持される
- 順序変更は未割り当て Task のみを対象とする(Cycle 割当済みは並べ替え対象外)
- 並べ替えの結果が即時に一覧へ反映される
