# US-27: ステップ定義(パイプライン)をカスタマイズする

## メタ
- 親: [s1/index.md](./index.md)
- 対応 S2 画面 (確定後に追記): SCR-XX
- ステータス: 確定

## 3 観点

### なぜするか (Why)
S1〜S7 は現状の標準だが、PJ や手法によっては別構成のパイプラインが要る。ステップ定義を固定にすると方法論の進化を取り込めない。可変点を残すことで将来の手法変更に耐える。

### UX へのインパクト
ユーザーは(優先度低だが)ステップの並びや構成を別パイプラインに差し替えられる。標準の AI-DLC 以外の進め方にもスタジオを適用できる余地が生まれる。

### 受け入れ条件 (AC)
- [v0.0.x] ステップ定義(パイプライン構成)を標準の S1〜S7 以外に変更できる
- 変更したパイプライン定義は PJ 単位で保存される
- Cycle 実行が選択中のパイプライン定義に従って進む
- 不正なパイプライン定義(空/循環等)は保存時に拒否される
- 標準パイプラインへいつでも戻せる
