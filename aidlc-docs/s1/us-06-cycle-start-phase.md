## 3 観点

### なぜするか (Why)
Cycle を作っただけでは何も動かない。サイト操作で AI-DLC のフェーズ(S1 など)を headless に起動できて初めて、人間が IDE を触らず AI に開発を回させる体験が成立する。MVP の心臓部。

### UX へのインパクト
ユーザーは「開始」を押すだけで、ローカル常駐の AI が該当 Phase を裏で走らせ始める。端末やコマンドを一切叩かずに開発工程が動き出す。

### 受け入れ条件 (AC)
- [v0.0.1] 開始可能な Cycle に対し、サイト操作で先頭 Phase を headless 起動できる
- 起動後、Cycle のステータスが「AI 実行中(Phase 名)」に遷移する
- Phase 起動は Claude Agent SDK 経由でローカル常駐プロセスに委譲される
- 起動失敗(プロセス未起動等)時はエラーが表示され Cycle は実行中にならない
- 起動済み Phase の現在状態(進行中/Q待ち/レビュー待ち等)が画面で識別できる
