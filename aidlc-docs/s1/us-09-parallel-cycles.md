## 3 観点

### なぜするか (Why)
1 つの Cycle が人間判断待ちで止まっている間も、別の Cycle を進められれば開発スループットが上がる。git worktree で作業空間を分離すれば、互いを汚さず同時進行できる。

### UX へのインパクト
ユーザーは複数の開発ラインを同時に走らせ、片方が Q 待ちでも他方のレビューに回せる。待ち時間の遊びが減り、人間は常にどこかの判断に従事できる。

### 受け入れ条件 (AC)
- [v0.0.x] 2 つ以上の Cycle を同時に実行状態にできる
- 各 Cycle は独立した git worktree 上で動作し、作業が互いに干渉しない
- 各 Cycle の進行状態が一覧で並べて把握できる
- 並行数に応じてリソース競合(同一ブランチ等)が起きないことが保証される
