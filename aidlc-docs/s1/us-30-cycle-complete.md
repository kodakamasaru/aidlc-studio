# US-30: Cycle を完了(クローズ)する

## メタ

- ステータス: 確定

## 3 観点

### なぜするか (Why)
S7 まで終えた Cycle は明示的に締めて次へ渡す必要がある。完了処理がないと、終わった Cycle が Active に残り続け、Dashboard が散らかり、次サイクルへの引き継ぎ(ledger)が確定しない。

### UX へのインパクト
ユーザーは S7 まで通った Cycle を「完了」にして区切りをつけられる。完了済みは Active から外れ、含まれた Task はクローズ、次サイクルへの引き継ぎが確定する。

### 受け入れ条件 (AC)
- [v0.0.x] 全ステップ(S7 まで)を終えた Cycle を完了(クローズ)できる
- 完了時に含まれる Task が完了状態に更新される
- 完了 Cycle は Active Cycles から外れ、完了済みとして参照できる
- 完了時に次サイクルへの引き継ぎ(ledger)が確定・記録される
- 未完了ステップが残る Cycle はクローズできない(またはその旨が警告される)
