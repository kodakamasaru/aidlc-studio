## 3 観点

### なぜするか (Why)
headless 実行は API エラー・ハング・無応答で止まり得る。人間が IDE を開かない前提では、止まりを検知してサイトから再実行できないと Cycle が永久に詰まる。MVP の堅牢性の最低ライン。

### UX へのインパクト
ユーザーは「AI が止まっている」と画面で気づき、ボタン 1 つで再実行できる。端末でプロセスを覗いて手動再起動する必要がない。

### 受け入れ条件 (AC)
- [v0.0.1] AI 実行が一定時間応答しない(stall)と検知し、画面に「停止中」を表示する
- 停止中の Phase をサイト操作で retry(再実行)できる
- retry は当該 Phase を直近の安全な地点から再開し、成果物の二重生成を避ける
- retry 回数や直近のエラー要因がユーザーに提示される
