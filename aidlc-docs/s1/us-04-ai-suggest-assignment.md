## 3 観点

### なぜするか (Why)
Backlog が大きくなると、どの Task を束ねると 1 Cycle として筋が良いかの判断は重い。AI が依存関係や関連性を踏まえて束ね方を提案し、人間は判断に集中できる。

### UX へのインパクト
ユーザーは「提案して」を押すだけで、AI が複数の Cycle 案(どの Task をまとめるか)を提示する。気に入れば採用、違えば手動(US-03)で調整でき、ゼロから組む手間が消える。

### 受け入れ条件 (AC)
- [v0.0.x] 未割り当て Task 群に対し、AI が 1 つ以上の Cycle 束ね案を生成する
- 各提案には束ねる Task と束ねた理由(根拠)が添えられる
- ユーザーは提案を承認すると Cycle として確定でき、却下すると破棄できる
- 提案の採否は Decision として記録される
- 提案は叩き台であり、採用後も手動で Task を増減できる
