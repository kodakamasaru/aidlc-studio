## 責務 (1〜2 行)
**設定 >「デフォルトのステップ構成」**(全サイクル共通テンプレ)で、各ステップの中身を **AI と対話 → 変更案提案 → プレビュー → 承認して適用**で編集する。定義は **要約(人間向け)↔ 全文(AI 実行用)の 2 層**で保持。適用は既存 `customizePipeline` 経由で `pipelineDef` JSON に永続化(S3 反映 / S4 §3.7)。

## 外部依存
- **Unit-01**: 編集対象 = `StepContracts`(やること/入力/成果物/完了条件/人の確認/並行可否)。`pipelineDef`(`StepDef[]`)JSON に乗る。
- **Unit-06**: 設定画面は PageGuard/Comparator 抽出基盤の上に作る。
- 既存: `app/services/project-service.ts` の `customizePipeline`(再利用 / 新テーブルなし)・`infra/http/routes/projects.ts`・SCR-01(設定導線)。

## I/F 定義 (この Unit が公開する契約)

| 操作 | 入力 | 出力 | エラー |
|------|------|------|--------|
| 設定: デフォルトのステップ構成画面 | — | 各ステップの要約一覧(平易な「やること」必須) | — |
| 対話編集 | ユーザー要望(自然文) | AI 変更案 → プレビュー差分 → 承認 | 未承認は適用しない |
| PATCH pipelineDef | 承認済み変更 | `customizePipeline` で `pipelineDef` 更新・永続化 | `validatePipeline`(非空/id 一意)違反は err |
| 要約↔全文 2 層 | 要約変更 | 追従して全文(AI 実行用)を更新・保持 | — |

- 新規サイクルはこのテンプレを複製。サイクル側のステップ構成は **控えめ導線 + 進行中は読み取り専用**。

## 主な AC(US 由来)
- 設定に「デフォルトのステップ構成」画面(新規サイクルが複製)。
- 各ステップの中身を編集、編集は対話式(要望→提案→プレビュー→承認)。
- 定義は要約↔全文の 2 層、各ステップに平易な「やること」必須。
- 適用が永続化され次回 Run に反映。サイクル側は控えめ導線+進行中読み取り専用。
- E2E で「対話→提案→適用→実行反映」pass。
