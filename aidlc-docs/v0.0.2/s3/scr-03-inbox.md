## 状態網羅(= S7/S8 実装契約)
| state | 意味 | 主な視覚差分 |
|-------|------|-------------|
| `list` | 相談カード一覧 | **コンパクト**(種類 + ステップ名 + 1 行サマリ + 件数のみ) |
| `empty` | 相談なし | empty state |

## カード種別(平易な名称)
| 種類 | アイコン | 色 | 動詞 | カードに出す情報 |
|------|---------|----|------|----------------|
| 見送りの相談 ★新 | `!` | amber | 判断する | ステップ名 + 「N 件の項目を見送りたい」 |
| 質問 | `?` | indigo | 回答する | ステップ名 + 「質問が N 件あります」 |
| できあがりの確認 | `◎` | purple | 確認する | ステップ名 + 短い案内文 |

## レイアウト / 視覚意図(フィードバック反映)
- **カードはコンパクト**: 質問本文・選択肢・gap 全文は **出さない**。1 ステップで質問が複数出る前提なので、カードには **件数**だけ示し、中身は開いてから(SCR-05)確認する。
- **見送りの相談**は「AI が理由つきで申請したもの」だけが届く(理由なしの抜けは AI が自動で作り直すため受信箱には出ない)。カードには「N 件の項目を見送りたい」と要約のみ。
- 種類は色 + アイコン + 動詞の三重符号で識別(色覚多様性に配慮)。
- 未処理件数は sidebar に常時表示。要対応(見送りの相談など)があるときは件数バッジを amber に。

## interaction
- カード全体がリンク。質問/見送り → SCR-05、できあがりの確認 → SCR-04。
- カード上では即時アクションを出さない(中身を見てから判断)。
- 件数は push(SSE)でリアルタイム更新。

## a11y
- カードは `role="link"`、`aria-label` に種類 + ステップ名 + 件数。
- 種類は色のみに依存しない(アイコン文字 + 動詞)。

## motion(文字で)
- hover: 背景 150ms ease-out。新着: 上から 200ms cubic-bezier(0.16,1,0.3,1) で fade + slide-down。

## native 固有挙動でドメイン側に影響しそうな項目
- **相談(HumanTask)の種類**に「見送りの相談(descope)」を追加。ただし発火条件は「AI が理由つきで申請したとき」のみ(SCR-02 参照)。
- カードは **要約 + 件数**だけ持てばよい(本文は詳細画面で取得)。1 ステップ複数質問に対応するため、質問はステップ単位でまとめ、件数を持つ。

## 3 巡目の更新(フィードバック)
- 受信箱は **横断のお知らせ一覧**(判断はしない)。各カードは 種類 + サイクル名/ステップ名 + 件数 + 「サイクルで対応 ›」のみ。アクション動詞(回答する/判断する)は外し、クリックで該当サイクルへ遷移してサイクル側で対応する。

## この画面固有の 質疑応答ログ

### Q-01 — Inbox に何を表示するか
- ユーザー指摘「1 ステップで複数質問が出るので、Inbox に質問内容まで出すのは厳しい」。→ カードは件数 + 要約のみ、本文は SCR-05 で(D-01)。

---

## この画面固有の AI が独自に決めたこと と 理由(フィードバック反映)

### D-01 — カードはコンパクト(質問本文・選択肢を出さない / 件数のみ)
- **理由**: ユーザー指摘どおり、1 ステップで複数質問が出ると一覧が破綻する。一覧では「どのステップで・何件・どの種類」だけ把握し、中身は開いてから確認する。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

### D-02 — 受信箱に届く見送りは「AI が理由つきで申請したもの」だけ
- **理由**: ユーザー指摘「明示的な申請なしの gap は AI レビューで差し戻し」。理由なしの抜けで受信箱を埋めない。届く見送りカードは必ず AI の理由を伴う。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

## この画面固有の 棄却した案

### R-01 — カードに質問文 + 選択肢プレビューを全部出す(初版)
- **棄却理由**: 1 ステップ複数質問で一覧が縦に伸びすぎる。件数表示に変更(D-01)。

### R-02 — gap 全件をカードにインライン展開(初版)
- **棄却理由**: 同上 + 新モデルでは「理由つき見送り申請」だけが届くため、カードは要約で足りる。
