## 状態網羅(= S7/S8 実装契約)
| state | 意味 | 主な視覚差分 |
|-------|------|-------------|
| `question` | 質問への回答 | 質問 echo + 選択肢(最後に「その他=自由入力」)。**入力欄は1個** |
| `descope` | 見送りの相談への判断 | AI の理由 + 選択肢(やる / 見送る / 前のステップからやり直す / その他) |
| `confirm` | 「見送る」の確認 | scrim + dialog(不可逆操作の再確認) |
| `resume` | 回答後の処理中 | spinner + progress + 自動遷移予告 |

## 回答 UI(フィードバック反映)
- 選択肢はラジオ(単一選択)。**最後の選択肢が「その他(自由に回答する)」**で、選ぶと **その場に入力欄が1個だけ**現れる。
- 「補足コメント欄 + 自由入力欄」の二重 textarea は廃止。入力は常に1個に統一。
- 1 ステップで質問が複数ある前提なので、上部に「質問 1 / 3」のように進捗を出し、1件ずつ回答する。
- AI のおすすめには控えめなバッジ(`AI のおすすめ`)。

## 見送りの判断(descope)の選択肢(フィードバック反映)
AI が理由つきで「見送りたい」と申請したものに対し、人間が選ぶ:
1. **やっぱり今回つくってもらう** — 見送らず、このステップで作り直す
2. **今回は見送る(次のバージョンへ)** — 持ち越しに記録(確認ダイアログあり / 不可逆)
3. **前のステップからやり直す ★新** — 原因が前段にありそうなとき、どのステップから戻すか選ぶ
4. **その他(自由に回答する)** — 入力欄1個で自由指示

## レイアウト / 視覚意図
- 質問 → 選択肢 → 送信 の縦フロー。`content-narrow`。
- descope は先頭に AI の理由(`.gap-box`)、続いて4択。「前のステップからやり直す」を選ぶと、戻り先ステップの選択(`.select`)が現れる。
- 「見送る」は不可逆 → confirm ダイアログ(danger 確定)。
- resume: 処理中を明示し、完了後に進行状況へ自動遷移。

## interaction
- 選択肢クリックで選択。「その他」選択で入力欄出現。「前のステップからやり直す」選択で戻り先 select 出現。
- 送信: `Cmd/Ctrl+Enter`。「見送る」選択時のみ confirm。
- 質問は 1 / N をめくって回答。全件回答で resume。

## a11y
- 選択肢は `role="radiogroup"`。出現する入力欄/セレクトは選択時に focus 移動。
- confirm は `role="dialog" aria-modal`、focus trap、`Esc` で閉じる。
- resume は `role="status" aria-live="polite"`。

## motion(文字で)
- 「その他」入力欄 / 戻り先 select 出現: 150ms ease-out で height + fade。
- confirm: scrim 150ms fade + dialog 200ms slide-up。

## native 固有挙動でドメイン側に影響しそうな項目
- **回答ペイロード**: 「選択肢 id」または「その他=自由入力テキスト」(入力は1本)。S5/S6 の回答モデルをこの形に。
- **見送りの判断結果(4種)**: つくる(差し戻し)/ 見送る(持ち越し)/ 前のステップからやり直す(戻り先ステップ id 付き)/ 自由指示。「やり直す」は **戻り先ステップ** をデータに持つ。
- 質問はステップ単位で複数件。1 件ずつ回答 → 全件回答で resume。

## 3 巡目の更新(フィードバック)
- 本画面は **サイクル文脈**(パンくず: サイクル / バージョン / ステップ。受信箱からでなくサイクルから到達。戻り先もサイクル詳細)。受信箱はお知らせ一覧に徹する。
- 見送り判断の「前のステップからやり直す」に **AI の推奨ステップ(★)+ 推奨理由** を表示(戻り先を人間任せにしない)。
- 推奨は **固定ではなく、今回の抜けの原因をたどって算出**する(例: 抜けの大元が要件段階なら「要件ヒアリング」を推奨)。原因分析の結果を理由として明示し、人間は別ステップも選べる。S5/S6 で「gap → 原因のあるステップ」を導く責務を AI に持たせる。

- ユーザー指摘「選択肢に OTHER を入れて入力欄は1個」。→ 「その他=自由入力」を選択肢化、入力欄1本に統一(D-01)。

### Q-02 — 見送り時の選択肢
- ユーザー指摘「スコープ落としの提案に『ステップ XX からやり直す』がないのは変」。→ 4 択に「前のステップからやり直す」を追加(D-02)。

## この画面固有の AI が独自に決めたこと と 理由(フィードバック反映)

### D-01 — 「その他(自由入力)」を選択肢に入れ、入力欄は1個に統一
- **理由**: ユーザー指摘どおり。補足 + 自由入力の二重欄は煩雑。選択肢の一つを「その他」にし、選んだときだけ入力欄を1個出す方がすっきりする。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

### D-02 — 見送りの判断に「前のステップからやり直す」を追加
- **理由**: ユーザー指摘どおり。原因が前段にあるときは、見送る/作るの二択でなく「戻ってやり直す」が必要。戻り先ステップを選べるようにした。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

### D-03 — 「見送る」のみ確認ダイアログ(不可逆のため)
- **理由**: 見送りは持ち越しへの記録で不可逆的意味合い。danger 確定で誤操作を防ぐ。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

### D-04 — 質問は「1 / N」で1件ずつ回答
- **理由**: 1 ステップで複数質問が出る前提(SCR-03 と整合)。一度に並べず1件ずつ確実に答える。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

## この画面固有の 棄却した案

### R-01 — 補足コメント欄 + 自由入力欄の二重 textarea(初版)
- **棄却理由**: 入力欄が2つで煩雑。「その他」選択肢 + 単一入力に統合(D-01)。

### R-02 — 見送りを「つくる / 見送る / 後回し」の3択に固定(初版)
- **棄却理由**: 「前のステップからやり直す」が抜けていた。4 択に拡張(D-02)。
