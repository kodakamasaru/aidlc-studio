# SCR-03: 受信箱(更新) — v0.0.2

## メタ
- 親: [s2/index.md](./index.md)
- 対応 US: [US-03](../s1/us-03-completeness-gate.md), [US-08](../s1/us-08-qa-resume-loop.md)
- ステータス: 確定

## 目的

v0.0.1 の受信箱(InboxPage)に **descope カード種別** を追加し、Q カードに **AI 選択肢のプレビュー** を表示する。ユーザーは受信箱から descope の承認/差し戻しができる。

## 主要要素

### A. 新カード種別: descope (US-03)

v0.0.1 の card kinds(`question`, `visual_review`, `device_check`, `decision`, `backtrack`, `stall_retry`)に `descope` を追加。

- **バッジ**: `!` amber (注意喚起の意味)
- **アクション動詞**: 「判断する」
- **メタ情報**: cycle / step / gap 数

### B. Q カード拡張: 選択肢プレビュー (US-08)

Q カードに AI が提示した選択肢のサマリーを表示。

- 選択肢の数と推奨マークをカード上にプレビュー
- カード全体がリンク(既存動作と同じ)

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — descope バッジは amber(警告色)にする
- **理由**: descope は「要件が漏れている」という異常状態。他の Q(indigo)や レビュー(purple)より注意喚起度が高い。amber は v0.0.1 の stall バッジと同じ色だが、アイコンを `!` にして区別。
- **判断**(ユーザー記入): 承認 | 上書き | 保留
- **上書き内容**(上書き時のみ):

### D-02 — descope カードは gap 全件を表示する(S2 Q-03 で確定)
- **理由**: gap の見落としは品質ハーネス原則(漏れ無し)に反する。件数省略すると重要な未対応要件が隠れる。カードが長くなっても全件列挙を優先。Q カードの選択肢プレビューも同様に全件表示する。
- **判断**(ユーザー記入): 承認 | 上書き | 保留
- **上書き内容**(上書き時のみ):

## この画面固有の 棄却した案

### R-01 — descope を既存 `decision` kind で表現する
- **棄却理由**: decision は一般的な意思決定。descope は「要件漏れ」という特定の品質問題であり、区別が必要。hard gate の文脈を表現するには専用 kind が適切。

### R-02 — カード上で直接 descope 承認ボタンを出す
- **棄却理由**: カード一覧での即時アクションは誤操作リスクがある。SCR-05 で gap 内容と選択肢を確認してから判断するのが安全。
