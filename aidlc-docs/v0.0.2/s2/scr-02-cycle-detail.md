# SCR-02: サイクル詳細(更新) — v0.0.2

## メタ
- 親: [s2/index.md](./index.md)
- 対応 US: [US-02](../s1/us-02-engine-pipeline.md)
- ステータス: 確定

## 目的

v0.0.1 のサイクル詳細画面に、**gen→eval パイプラインの状態可視化**を追加する。ユーザーは AI の実行状況(gen/eval のどちらが動いているか)をリアルタイムに把握できる。

> 注: Step定義エディタは SCR-01(サイクル一覧)に配置(index.md D-01)。本画面は実行状況の表示に専念する。

## 主要要素

### A. PhasePipeline 拡張 (US-02)

v0.0.1 の S1-S7 パイプライン表示に role バッジと中間状態を追加。

- **role バッジ**: アクティブな Phase ノードに `GEN` または `EVAL` の小バッジを表示
- **中間状態表示**: deterministic gate 実行中は短いプログレスバー
- **状態名の変更**: Run Panel の状態名をパイプライン状態一覧(index.md 参照)に更新

### B. Run Panel 拡張 (US-02)

v0.0.1 の Run Panel(state-dependent rendering)に新しい状態パネルを追加。

- **GeneratorRunning**: ログストリーム + role バッジ `GEN`
- **DeterministicGate**: プログレス「自動チェック中...」(1-3秒)
- **GateFailed**: 失敗理由 + 「再実行」ボタン + 受信箱に HumanTask(stall_retry)も生成
- **EvaluatorRunning**: ログストリーム + role バッジ `EVAL`
- **EvalDone_Fail**: gap 検出メッセージ + 「受信箱を確認」リンク

## この画面固有の 質疑応答ログ

### Q-01 — gate 失敗時の「再実行」は SCR-02 と受信箱のどちらからでも可能にするか？
- **回答**(ユーザー記入):
  > 両方から可能。
- **確定**(AI 記入):
  > SCR-02 の「再実行」ボタンと、受信箱の HumanTask(stall_retry)の両方から retry できる。どちらかで retry すると、もう片方の表示も解決済みに更新される(状態同期)。S3 でこの同期挙動を設計する。

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — deterministic gate 失敗は自動リトライなし(手動のみ)
- **理由**: gate 失敗は「成果物に必須 block が欠けている」という異常状態。自動リトライしても同じ結果になる可能性が高い。ユーザーが状況を把握してから再実行する方が安全。受信箱に HumanTask(stall_retry)を出して気付けるようにする(index.md Q-02)。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

## この画面固有の 棄却した案

### R-01 — PhasePipeline に gen/eval の2ノードを表示する
- **棄却理由**: S3 は1つの Phase であり、gen と eval は内部状態。Phase を2つに見せると混乱する。role バッジで表現する方が正確。
