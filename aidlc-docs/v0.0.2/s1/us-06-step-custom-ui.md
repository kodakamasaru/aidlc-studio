## 3 視点

### なぜするか (Why)
Step の契約(Output / Verification / HumanGate / Escalation)をコードではなく **画面から編集** できるようにする。各プロジェクトで Step の振る舞いをカスタマイズ可能にし、「Step は固定」という制約を外す。これが Step 可変化の UI 面。

### UX へのインパクト
ユーザー(開発者)はサイクル詳細画面(SCR-02)から Step の契約を直接編集できる。コードを触らずに「この Step は人間ゲートを通す」「この Step の成果物 Profile を変える」を設定でき、次回実行に即座に反映される。

### 受け入れ条件 (AC)
- [ ] Step 定義編集フォームが SCR-02(サイクル詳細)に追加される
- [ ] 4つの契約タイプ(Output / Verification / HumanGate / Escalation)を編集できる
- [ ] execMode(sequential / parallel)を切り替えられる
- [ ] 成果物 Profile の block 構成を閲覧・編集できる
- [ ] 編集結果が API 経由で永続化される
- [ ] 次回 Run 実行時に更新された StepDef が反映される
- [ ] 既存 Step(契約なし)も表示され、空状態が明示される
- [ ] E2E テストで Step 編集→実行反映フローが pass する
