> **S3 からの反映(手戻り)**: ①編集の置き場所を「サイクル詳細(SCR-02)」から **「設定 > デフォルトのステップ構成」**(全サイクル共通テンプレ)に変更。サイクル側は確認中心の控えめ導線。②編集方式を **手入力フォームから「AI と対話 → AI が変更案を提案 → プレビュー → 承認して適用」** に変更。③ステップ定義は **「要約(人間向け)↔ 全文(AI 実行用)」の2層**で持つ(UI は要約、AI は全文で実行)。詳細は [s3/scr-01](../s3/scr-01-cycle-list.md)。

## 3 視点

### なぜするか (Why)
Step の振る舞い(やること / 完了の条件 / 人の確認 / 並行可否 = 内部の Output / Verification / HumanGate / Escalation を平易化したもの)を、コードでなく **画面から調整** できるようにする。「Step は固定」の制約を外す。**全サイクル共通のひな形(デフォルト)は設定に常設**し、サイクル作成時に複製される。

### UX へのインパクト
ユーザー(開発者)は **設定の「デフォルトのステップ構成」** で、各ステップの中身を **AI に相談しながら**調整できる(手入力フォームではなく対話 → 提案 → プレビュー → 承認)。画面に出るのは人間向けの **要約**で、実際に AI が使う指示はそれを基に展開した **全文**。サイクル側ではステップ構成を控えめなリンクから確認でき、開始後は読み取り専用。

### 受け入れ条件 (AC)
- [ ] **設定に「デフォルトのステップ構成」画面がある**(全サイクル共通のひな形 / 新規サイクルはこれを複製)
- [ ] 各ステップの中身(やること / 入力 / 成果物 / 完了の条件 / 人の確認 / 並行可否)を編集できる
- [ ] 編集は **対話式**(ユーザーが要望 → AI が変更案を提案 → プレビューで差分確認 → 承認して適用)
- [ ] ステップ定義は **要約(人間向け表示)と 全文(AI 実行用)の2層**で保持され、要約から全文を確認できる
- [ ] 各ステップに「やること」の平易な説明が必ず付く(役割が一目でわかる)
- [ ] 適用結果が永続化され、次回 Run 実行時に反映される
- [ ] サイクル側のステップ構成は控えめな導線で、進行中は読み取り専用
- [ ] E2E テストで「対話 → 提案 → 適用 → 実行反映」フローが pass する

## この US 固有の AI が独自に決めたこと と 理由

### D-01 — Step 編集フォームを SCR-02(サイクル詳細)に配置する 〔S3 で上書き〕
- **理由**: Step は Cycle の子要素。Cycle 詳細画面に配置するのが自然。
- **判断**(ユーザー記入): 上書き(S3)
- **上書き内容**: デフォルトのステップ構成は **「設定」** に常設(全サイクル共通テンプレ)。サイクル側は確認中心の控えめ導線。配置を SCR-02 から変更。

### D-02 — 編集は手入力でなく「対話 → AI 提案 → プレビュー → 承認」(S3 反映)
- **理由**: 4 契約タイプの手入力は内部概念を人間に押し付ける。やりたいことを言葉で伝え、AI が定義に翻訳し、人間は結果を見て承認するだけにする。
- **判断**(ユーザー記入): 承認 | 上書き | 保留

### D-03 — ステップ定義は「要約(人間向け)↔ 全文(AI 実行用)」の2層(S3 反映)
- **理由**: 画面に出す要約だけを AI のコンテキストにすると粒度不足で遂行できない。人間は要約を見て判断、AI は要約を展開した全文で実行する。要約変更に追従して全文を更新。全文の生成・保持は S4(技術仕様)/ S5 で確定。
- **判断**(ユーザー記入): 承認 | 上書き | 保留
