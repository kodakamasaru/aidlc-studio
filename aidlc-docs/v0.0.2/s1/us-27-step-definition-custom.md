## 3 観点

### なぜするか (Why)
品質ハーネスは各 step の契約(Output/Verification/HumanGate/Escalation)に依存する(§2)。契約を画面から編集できることで、PJ ごとに品質基準を調整でき、ハーネスの価値が最大化する。StepDef 拡張(A)の UI 面。

### UX へのインパクト
ユーザーはパイプライン画面で step の契約フィールドをフォーム編集し、成果物 Profile(必須 block 集合)を確認できる。次回実行から即反映。

### 受け入れ条件 (AC)
- [v0.0.2] ステップ定義(パイプライン構成)を標準の S1〜S7 以外に変更できる
- 変更したパイプライン定義は PJ 単位で保存される
- Cycle 実行が選択中のパイプライン定義に従って進む
- 不正なパイプライン定義(空/循環等)は保存時に拒否される
- 標準パイプラインへいつでも戻せる
- [v0.0.2] StepDef の契約(Output/Verification/HumanGate/Escalation)を UI から編集できる
- [v0.0.2] 編集した契約は次回実行に反映される
- [v0.0.2] 成果物 Profile(必須 block 集合)が Step に紐づいて表示される
