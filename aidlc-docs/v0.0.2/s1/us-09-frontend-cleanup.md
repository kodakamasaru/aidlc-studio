## 3 視点

### なぜするか (Why)
v0.0.1 で画面ガード(権限チェック)と一覧ソートが各画面に重複実装されている。US-06/US-07 で新規画面要素を追加する前に共通化しておかないと、重複がさらに増える。リファクタは後回しにするほど負債が増えるため、機能拡張(P5)の前に実施する。

### UX へのインパクト
ユーザー(開発者)への直接的な機能変化はない。間接的に、画面遷移の一貫性が保たれ、バグ修正のコストが下がる。

### 受け入れ条件 (AC)
- [ ] PageGuard 抽象: 各画面の権限チェックロジックが共通フック/コンポーネントに抽出される
- [ ] 一覧 Comparator 共通: 一覧画面のソートロジックが共通ユーティリティに抽出される
- [ ] 既存画面(SCR-01〜SCR-05)の動作が変わらない(リグレッションテスト全 pass)
- [ ] 抽出後の各画面コードが重複なしになる
- [ ] 既存テストが全 pass
