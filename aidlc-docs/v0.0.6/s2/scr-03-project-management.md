# SCR-03: プロジェクト 作成 / 切替 / リセット

## メタ
- 親: 画面要素の一覧
- 対応 US: US-12(プロジェクト作成/リセット/切替 UI + legacy 正規化)
- ステータス: 確定

## 目的
人間が IDE やファイル操作なしで、AI-DLC を回す対象プロジェクトを作成・切替・リセットできる入口。現状の「最初のプロジェクト固定」を解消し、自分の任意リポを選べるようにする。

## 主要要素
- **プロジェクト一覧**: 名前 + 対象リポ(人間が指定したパスの表示名)+ 現在サイクル + 状態。
- **切替**: クリックで対象プロジェクトを切替(ボード/Inbox がそのプロジェクトに切り替わる)。
- アクション: `新規作成`(リポを指定して登録)/ `リセット`(確認付き)/ 切替。
- **legacy 正規化バナー**: 旧形式のプロジェクトを検出したら「移行が必要」を出し、ワンクリックで安全に正規化(backup を取ってから)。

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — legacy 正規化は「検出 → バナー → ワンクリック移行(backup 付き)」の導線にする
- **理由**: US-12 の AC「安全な正規化マイグレーション(データを壊さない / backup)」。人間にマイグレーションの内部を理解させず、検出と実行だけを画面に出す(責務契約①)。破壊操作なので確認 + backup を必須にする。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

### D-02 — 対象リポは「表示名 + 人間が選んだリポ」で持ち、内部の絶対パスは画面に出さない
- **理由**: 責務契約①(内部秘匿 / パス露出禁止)。人間には表示名で見せ、repoPath は内部で US-10 の reconcile/ledger に渡す。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

## この画面固有の 棄却した案

### R-01 — リセットを確認なしの即時実行にする
- **棄却理由**: 破壊操作(サイクル/成果物の消去)。確認 + できれば backup を必須にする(取り返しのつかない操作の保護)。
