# SCR-01: 自走ボード(自走状態バッジ + 稼働中タスク一覧)

## メタ
- 親: 画面要素の一覧
- 対応 US: US-01(スケジューラ)/ US-05(reconcile-resume)/ US-06(stall)/ US-07(逐次監視)/ US-08(稼働台帳)
- ステータス: 確定

## 目的
既存ボードに「スケジューラが今どう自走しているか」を一目で足す。人間は実行中・待ち・parking・自動復帰中を区別でき、放置していても状況を掴める。

## 主要要素
- **稼働中タスク一覧**(US-08 稼働台帳の描画): どのサイクルのどのステップが起動中か + 最終活動からの経過(US-07 逐次監視の last-activity)。
- **自走状態バッジ**(各ステップ): `実行中` / `backoff待ち`(上限/レート) / `parking`(human-gate 回答待ち) / `stall→自動retry`(無音 timeout) / `resume復帰`(再起動後の継続)。
- **並列数インジケータ**: 現在の起動数 / 上限 N(常に N 以下)。
- アクション: 個別タスクの詳細へ遷移(既存の会話スレッド/レビューへ)。retry 手動トリガは残す(逃げ道)。

## この画面固有の 質疑応答ログ

### Q-01 — 状態バッジの粒度(5 種)で過不足ないか
- **回答**(人間の回答を AI が記入):
  > 5 種で OK(2026-06-21)。
- **確定**(AI 記入):
  > 実行中 / backoff待ち / parking / stall→retry / resume復帰 の 5 バッジで確定。

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — 状態を 5 バッジ(実行中 / backoff待ち / parking / stall→retry / resume復帰)に整理
- **理由**: 設計の自走状態(§4〜§8)を人間が区別する必要があるのは「動いている / 時間で回復する待ち / 人間待ち / 自動復旧中 / 再起動継続」の 5 つ。内部の細かな遷移(検証 NG の作り直し回数等)は技術詳細でバッジに出さず、retry カウンタは補助表示に留める。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

### D-02 — backoff待ちは「約 N 分後に自動再開」と人間語で出す
- **理由**: 上限/レートは時間で回復する待ち。人間に行動を求めない(routine human-gate でない)ことを文言で明示し、不要な介入を誘発しない(責務契約②)。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

## この画面固有の 棄却した案

### R-01 — 稼働中タスクに内部 runId / pid / session_id を表示する
- **棄却理由**: サーバ内部情報の秘匿(責務契約①)。人間にはサイクル名・ステップ名・経過時間という事業語だけ見せる。
