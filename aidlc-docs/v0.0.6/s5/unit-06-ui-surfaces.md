## この Unit 固有の AI が独自に決めたこと と 理由

### D-01 — 2 画面を 1 Unit にまとめ、内部で分割
- **理由**: index D-05。どちらも「既存 store/CLI を読む presentation 面」で同層。依存先(US-12→Unit-05 / US-13→Unit-02/03 のデータ)が揃う Phase 3 が自然。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

### D-02 — legacy 正規化は backup 必須 + 確認付き(破壊操作ガード)
- **理由**: US-12 AC / SCR-03 D-01。マイグレーション・リセットは取り返しのつかない操作。backup と確認を必須にする。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

### D-03 — メトリクスは store の事実から導出し、人手の二次入力を挟まない
- **理由**: US-13 AC / SCR-04 D-02。手入力を挟むと振り返りが主観に戻る。Unit-02/03 が記録する件数も自走の信頼性指標として同レポートに含める。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし
