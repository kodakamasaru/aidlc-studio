## 3 視点

### なぜするか (Why)
web にプロジェクトの作成/リセット/切替の UI が無く、現状は最初のプロジェクト 1 つに固定されている。任意の自分のリポを対象に AI-DLC を回すには、人間がサイトからプロジェクトを選び・作り・リセットできる入口が要る。US-10(reconcile/ledger の project パラメータ化)が backend を任意 PJ 対応にするのと対になる UI 側。あわせて step-model-v2 以前の旧構造(S2.5 入り)の legacy プロジェクトを安全に正規化するマイグレーションが必要(古い形のまま残ると新パイプラインで誤動作する)。

### UX へのインパクト
ユーザーはサイト上でプロジェクトを作成・切替・リセットでき、IDE やファイル操作なしで「どのアプリのサイクルを回すか」を選べる。古い形式で残っていたプロジェクトも、壊さずに新構造へ移行されて一覧に正しく出る。

### 受け入れ条件 (AC)
- web から **プロジェクトの作成 / リセット / 切替** ができる(現状の固定を解消)。
- 旧 step-model(S2.5 入り / step-model-v2 前)の legacy プロジェクトを **安全に正規化するマイグレーション**(データを壊さない / backup を取る)。
- 一時 cleanup script + backup の整理を本 US に同送して片付ける(v0.0.5 の dead-code 削除はコードのみだった残務)。
- US-10(project パラメータ化)と整合: UI で選んだプロジェクトの repoPath が reconcile/ledger に正しく渡る。
