# US-06: stall 検知(timeout / claude 非依存)+ late-emit 冪等化 — 沈黙と遅延報告で不整合を起こさない

## 3 視点

### なぜするか (Why)
オーケストレーションの整合性は claude に一切依存させてはいけない(設計 §7)。claude の自己申告 stalled は早期ヒントに過ぎず、最終 backstop は claude 非依存の timeout でなければならない。また孤児の子プロセスが後から完了報告(late emit)してきたときに不整合化すると、再起動復帰が壊れる。今は late-emit がログ止まり(RunNotFound)で冪等化されていない(設計 §11)。

### UX へのインパクト
ハングして無言になったタスクも、一定時間で「止まった」と判定されて自動 retry される(永遠に待たされない)。過去に死んだはずの run が後から完了報告を出しても、ボードの状態が壊れない。ユーザーから見て「固まったまま放置」も「幽霊更新」も起きない。

### 受け入れ条件 (AC)
- **idle / 壁時計 timeout(claude 非依存)** で stall を検知し、retry に回す(設計 §5/§7-3)。idle は US-07 の逐次監視 + US-08 の last-activity が供給する(塊読みでは作れない)。claude の自己申告 stalled は早期ヒント扱いで、最終判定は timeout が握る。
- ハング(無出力)を idle timeout で stall 化 → retry(設計 §8)。
- **遅延 emit(孤児の子の後追い完了報告)を冪等に無視**する(`RunNotFound` で不整合化しない / 設計 §8/§11)。
- 二重 emit が来ても不整合を生まない(全遷移は原子コミット + 冪等 / 設計 §2)。

## この US 固有の AI が独自に決めたこと と 理由

### D-01 — stall 検知は claude の自己申告 stalled でなく timeout を権威にする
- **理由**: 設計 §7-3。自己申告はヒントに過ぎない。claude 非依存の timeout が最終 backstop でないと、claude がハングしたまま黙ったときに永久に検知できない。
- **種別**: 技術判断(AI 自走で確定)

## この US 固有の 棄却した案

### R-01 — late-emit を受けたら run を復活させて状態更新する
- **棄却理由**: 設計 §8「冪等に無視」。死んだ run の後追い報告で状態を書き換えると、reconcile で確定した正しい続き(US-05)を壊す。無視が正解。
