## 3 視点

### なぜするか (Why)
v0.0.2 では「理由なし gap(なぜ欠けたか説明できない欠落)」を loud(eval run を stalled にして理由提示)で代替した。だが理由のない欠落は人間の判断材料にならないので、本来は人間に出さず silent に自動再生成すべき(legacy S8-Q02 の残)。これは US-02(検証→自動 retry)と同思想を、ドメイン成果物の silent 再生成 UX に適用するもの。

### UX へのインパクト
ドメイン成果物に理由のない欠落があっても、ユーザーには loud な stall 通知が出ず、裏で黙って作り直される。Inbox に出るのは「理由のある判断依頼」だけになり、ノイズが減る。

### 受け入れ条件 (AC)
- 理由なし gap を検知したら、**loud な stall でなく silent な自動再生成**で埋める(新ドメイン再生成コマンド)。
- silent 再生成は US-02 の自動 retry ループと整合する(検証 → NG なら作り直し、人間に出さない)。
- 理由のある gap(契約逸脱・要判断)は従来どおり loud(レビュー/受け入れ経路)に上げる(silent に握り潰さない)。
