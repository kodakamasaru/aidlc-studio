## a11y
- VoiceOver / TalkBack ラベル: 各棒グラフ行は `role="meter"` + `aria-valuenow` + `aria-valuemax` + `aria-label="[ゲート名] [件数] 件"` または `"[ステップ名] [分] 分 retry [N] 回"` を付与。棒グラフは装飾目的であり、値はテキストで常に表示される(色のみに依存しない)。
- focus order: ページ上部の見出し → 検出ゲート別パネル(読み上げのみ) → ステップ別所要パネル(読み上げのみ) → 自動復旧ボックス → Q&A サマリパネル → アクションボタン(「CSV 出力」→「この内容で振り返りを回す」)。
- 色コントラスト基準(WCAG AA): 棒グラフの fill は `--color-primary: #6366f1` と `--color-stalled: #f59e0b`(warn 系)。どちらも `--color-surface-3: #212129` の track 上で十分な視認性。自動復旧率テキスト `--color-done: #22c55e` vs `--color-surface: #131316` → 約 6.2:1(AA 合格)。
- 三重エンコードの根拠: 棒グラフは色(primary / amber-warn) + バーの長さ + 右端の数値テキストの 3 つで値を伝える。色覚特性があっても長さ + 数値で判別できる。warn バーは amber + 右端「retry N」テキストでも区別できる。

## motion

## この画面固有の AI が独自に決めたこと と 理由

### D-01 — 自動復旧ボックスを独立した横並び表示にする
- **理由**: backoff 自動再開 / stall→作り直し / 要対応(inbox 落ち)の 3 種は、Q&A 系と性質が異なる(人間介在なし / 純粋な自走指標)。2 カラムグリッドに収めず独立 row にして視覚的に階層を変える。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

### D-02 — 「設計外の介在点 0 件」を緑色で明示する
- **理由**: S2 D-01「人間の介在点が固定 4 つ(要件/画面/レビュー/改善)以外に増えない」の裏取り。0 件が合格状態なので緑表示で一目わかるようにする。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし

### D-03 — ステップ別所要のバーに retry 件数を右端に補助表示する
- **理由**: retry が多いステップは問題の集中箇所として振り返りの着眼点になる。retry 数はバーの色(warn = amber)+ テキスト補注の二重表示で目立たせる。
- **種別**: 技術判断(AI 自走で確定)
- **上書き**: なし
