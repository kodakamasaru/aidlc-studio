## 3 観点

### なぜするか (Why)
セルフホスト前提のスタジオでは、対象リポジトリのパスや使用モデルが環境ごとに異なる。絶対パスや特定モデルをハードコードすると他環境で動かない。環境変数で外出しすることが可搬性の前提。

### UX へのインパクト
ユーザーは自分の環境(リポの場所・使う LLM モデルなど)を設定で与えるだけで、コードを書き換えずスタジオを動かせる。別マシンへの持ち運びや設定変更が容易になる。

### 受け入れ条件 (AC)
- [v0.0.x] 対象リポジトリのパス・使用モデル等を環境変数で設定できる
- 起動時に必須の環境変数が欠けていれば明確なエラーで停止する
- ソース中に環境固有の絶対パス・モデル名がハードコードされていない
- 設定値が AI 起動(Agent/orchestrator)に反映される
