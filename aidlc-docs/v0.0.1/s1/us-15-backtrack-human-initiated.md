## 3 観点

### なぜするか (Why)
進行中に人間が「これは前工程の認識違いだ」と気づくことがある(例: S6 まで進んで UI 認識のズレに気づき S2 へ戻す)。人間が任意の時点で手戻りを起動できないと、誤った前提のまま無駄が積み上がる。

### UX へのインパクト
ユーザーは現在の工程から戻したい工程を選び、理由を添えて手戻りを要求できる。気づいた瞬間にやり直しを宣言でき、手戻りを「もったいない」と先送りせずに済む。

### 受け入れ条件 (AC)
- [v0.0.x] ユーザーは現在の Cycle で、任意の前ステップへの手戻りを起動できる
- 手戻り要求には戻す先ステップと理由の入力が必須
- 手戻り起動後、対象ステップ以降の進行が再実行対象になる
- 手戻りの起点・理由が手戻りイベント/Decision として履歴化される
- 戻る際の引き継ぎ要点(何が違ったか/暫定方針)を記録できる
