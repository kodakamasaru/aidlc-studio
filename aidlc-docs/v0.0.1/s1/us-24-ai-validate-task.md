## 3 観点

### なぜするか (Why)
Backlog には重複 Task や、起票後の状況変化で陳腐化した Task が紛れ込む。AI が妥当性をチェックできれば、人間は「やる価値が今もあるか」の判断に集中でき、無駄な Cycle を未然に防げる。

### UX へのインパクト
ユーザーは Task に「他と重複」「もう不要かも」といった AI の注意フラグを見て、整理・統合・破棄を素早く判断できる。Backlog の鮮度を保てる。

### 受け入れ条件 (AC)
- [v0.0.x] AI が Backlog の Task 間の重複候補を検知して提示する
- AI が起票後の状況変化(成果物/他 Task との関係)による陳腐化候補を検知する
- 各指摘には根拠(どの Task/成果物と照合したか)が添えられる
- ユーザーは指摘を受けて Task を統合/破棄/維持できる
- 妥当性確認は提案であり、最終判断は人間が行う
