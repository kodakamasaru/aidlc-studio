# US-21: サイクル間の確定持ち越し(ledger)漏れを確認する

## メタ
- 親: [s1/index.md](./index.md)
- 対応 S2 画面 (確定後に追記): SCR-XX
- ステータス: 確定

## 3 観点

### なぜするか (Why)
あるサイクルで確定した決定や引き継ぎ要点が次サイクルに反映されないと、考慮漏れが再発する。ledger による cross-cycle reconciliation を可視化し、持ち越し漏れを人間が検知できることが品質の保険になる。

### UX へのインパクト
ユーザーは「前サイクルで決めたのに今サイクルに反映されていない項目」を一覧で見つけ、取りこぼしを早期に拾える。サイクルをまたいだ整合を機械的に確認できる。

### 受け入れ条件 (AC)
- [v0.0.x] サイクル間の持ち越し項目(ledger エントリ)を一覧で確認できる
- 反映済み/未反映(漏れ候補)が判別表示される
- 各持ち越し項目から出典サイクルと対象サイクルへ辿れる
- 未反映項目を Task 化 or 手戻りに繋げられる
