## 状態網羅
- **default**: 質問カード(Q バッジ + 番号 + 経過 + 「worktree は回答待ちで一時停止中」)+ 質問本文 + 補足 + 文脈リンク(関連成果物 / 該当ステップ出力)。下に回答テキストエリア(複数行・mono)+ 送信。送信ボタンは回答入力で活性(S2)。

## 挙動(web / レスポンシブ)
- 単一カラム max-width ~760px。質問を上、回答入力を下に置き、上から下へ読んで答える線形フロー。
- **文脈リンク**: 該当 Cycle / ステップ / 関連 US・SCR へ。質問の背景(該当ステップ出力)を別タブ or インラインで確認可(S2 備考)。
- 送信 → 回答を渡して Run 再開(resume)→ SCR-02。送信後、当該 Q カードは Inbox から消える。

## a11y
- 質問本文は見出し階層 + 本文。文脈リンクはリンクとして列挙(`?` 装飾は `aria-hidden`)。
- テキストエリアに `label`(回答)を関連付け。送信不活性時は理由(回答未入力)を補助テキストで明示。
- フォーカス順: 文脈リンク群 → 回答テキストエリア → 送信。

## interaction(web pointer / keyboard)
- 回答は複数行・コードブロック可(mono フォントで貼り付けやすく / S2 備考)。
- 送信: ボタン or Cmd/Ctrl+Enter。空回答では不活性。
- 送信後は SCR-02(running 再開)へ遷移。

## motion(文字で)
- 送信時: ボタンが spinner 化(0.8s)→ 完了で SCR-02 へ 200ms クロスフェード遷移。
- テキストエリア focus: 境界が primary、focus リング(primary-soft 3px)150ms。

## 設計連携メモ(S3/S5 へ)
- 「回答 → resume」は orchestration が停止中 Run を再開する経路。Run は Q 待ちで一時停止し、回答(外部入力)を受けて続行 → S5 で Run の停止/再開状態と HumanTask(question)の解決を集約として定義すること。
